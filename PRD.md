# CampusLog PRD

## 1. 프로젝트 개요

CampusLog는 대학생이 프로젝트, 공모전, 인턴 등 다양한 활동 경험을 잊지 않게 기록하고 AI가 역량과 성과를 정리해 자기소개서·포트폴리오·면접 준비에 바로 참고할 수 있게 돕는 성장 기록 서비스입니다.

이 프로젝트는 커널아카데미 대학생 AI 크루 UNIKER 1기 팀 핏줄의 10주 MVP 개발 프로젝트입니다. 현재 구현 기준은 4~5주차까지 완성할 1차 MVP이며, 많은 기능보다 대학생이 경험을 기록하고 필요할 때 다시 꺼내 쓸 수 있는 핵심 흐름을 완성하는 데 집중합니다.

## 2. 해결하려는 문제

대학생들은 프로젝트, 공모전, 대외활동, 인턴, 수상, 블로그, GitHub 등 다양한 경험을 여러 곳에 기록하거나 아예 기록하지 않는 경우가 많습니다.

그 결과 자기소개서, 포트폴리오, 면접, 대외활동 지원 등을 준비할 때 자신의 경험을 다시 찾고 정리하는 데 많은 시간이 걸립니다. 시간이 지나면서 중요한 경험을 잊어버리거나, 어떤 경험을 어떤 상황에 활용해야 할지 판단하기 어려운 문제도 발생합니다.

## 3. 대상 사용자

CampusLog의 주요 사용자는 프로젝트, 공모전, 대외활동, 서포터즈, 인턴, 취업 등을 준비하는 대학생입니다.

특히 다음과 같은 사용자를 주요 대상으로 합니다.

- 활동 경험은 있지만 정리해둔 곳이 없는 대학생
- 자기소개서, 포트폴리오, 면접을 준비하며 어떤 경험을 써야 할지 고민하는 대학생
- 대외활동, 서포터즈, 인턴 등 지원 목적마다 필요한 경험을 다시 골라야 하는 대학생
- 자신의 활동 경험에서 역량, 성과, 키워드를 뽑아내는 데 어려움을 느끼는 대학생

## 4. 사용 상황

사용자는 평소 프로젝트, 공모전, 인턴, 대외활동 등의 경험을 CampusLog에 기록합니다.

이후 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문, 대외활동 지원 문항처럼 특정 상황이 생기면 CampusLog에 해당 내용을 입력합니다. AI는 저장된 경험 전체와 분석 결과를 참고해 현재 상황에 가장 적합한 경험 1개를 추천하고, 왜 적합한지와 어떻게 활용하면 좋은지 알려줍니다.

이를 통해 사용자는 흩어진 경험을 다시 찾는 시간을 줄이고, 지원 목적에 맞는 경험을 더 쉽게 선택할 수 있습니다.

## 5. MVP 목표

1차 MVP의 목표는 대학생이 자신의 활동 경험을 직접 기록하고, AI가 그 경험을 분석해 지원 상황에 맞게 다시 활용할 수 있도록 돕는 것입니다.

1차 MVP에서는 다음 흐름을 완성하는 데 집중합니다.

1. 사용자가 활동 경험을 기록합니다.
2. 사용자가 활동 경험 상세 화면에서 AI 분석 요청을 실행하면 AI가 해당 경험을 요약하고 핵심 역량 태그, 주요 성과, 키워드를 분석합니다.
3. 분석 결과는 특정 활동 경험에 연결된 기록으로 localStorage에 저장됩니다.
4. 사용자가 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문 등을 입력하면 AI가 저장된 경험 전체와 분석 결과를 참고해 가장 적합한 경험 1개를 추천하고 활용 방향을 제안합니다.

## 6. 개발 단계 및 저장/배포 전략

CampusLog는 아래 3단계로 개발합니다. 현재 문서와 구현 작업의 기본 범위는 1차 MVP입니다.

### 6.1 1차 MVP: 4~5주차까지

- 기술 기준: Next.js, Browser localStorage, `sampleExperiences.ts` 기반 공통 샘플 데이터, OpenAI API, Vercel 배포
- 구현 제외: 로그인 / 회원가입, DB, Supabase, Spring Boot, MySQL, AWS, 파일 업로드
- 핵심 목표: 경험 기록 / 분석 흐름과 활용 추천 흐름을 별도 진입 흐름으로 검증
- 경험 기록 / 분석 흐름: 메인 화면 → 활동 경험 목록 → 활동 경험 상세 → AI 분석 요청 → AI 경험 분석 결과 저장
- 활용 추천 흐름: 메인 화면 → AI 경험 추천 및 활용 → 자기소개서 문항 / 포트폴리오 작성 목적 / 면접 질문 입력 → 저장된 경험 전체와 분석 결과 기준으로 가장 적합한 경험 1개 추천 → 참고 문장 복사 / 추천 결과 저장
- 저장 전략: 사용자가 입력한 활동 경험과 분석 / 추천 결과는 브라우저 localStorage에 저장합니다.
- 샘플 데이터 전략: 공통 샘플 경험은 `sampleExperiences.ts`에 정리해 첫 화면, 빈 상태, AI 경험 추천 및 활용 흐름 검증에 사용합니다.

### 6.2 2차 MVP: UNIKER 10주차 최종 발표 전까지

2차 MVP는 1차 MVP 이후 확장 예정 단계입니다. 외부 사용자가 직접 체험 가능한 사용자 기반 MVP를 목표로 하며, 아래 기술을 검토합니다.

- Next.js on Vercel
- Next.js API Route 또는 Server Action
- Supabase Auth
- Supabase Postgres DB
- Supabase Storage
- 사용자별 활동 기록 저장
- 수료증 / 활동사진 등 첨부파일 저장
- 사용자의 활용 목적에 따른 1순위, 2순위, 3순위 경험 추천과 각 추천 이유 비교 UI 검토

### 6.3 3차 확장: UNIKER 프로젝트 종료 이후

3차 확장은 UNIKER 이후 백엔드 포트폴리오 및 실제 서비스 확장 버전입니다. 1차 MVP나 2차 MVP 구현 범위가 아니며, 아래 기술을 별도 단계에서 검토합니다.

- Next.js 프론트엔드 유지
- Spring Boot REST API 도입
- MySQL 또는 AWS RDS
- AWS S3 파일 저장
- Spring Security / JWT 등 백엔드 구조 고도화

## 7. MVP 핵심 기능

1차 MVP의 핵심 기능은 아래 3개로 제한합니다.

### 7.1 활동 경험 기록

사용자는 프로젝트, 공모전, 인턴, 대외활동 등 다양한 경험을 기록할 수 있습니다.

기록 항목은 다음 내용을 중심으로 구성합니다.

- 제목
- 기간
- 역할
- 내용
- 성과
- 관련 링크

이 기능은 AI 분석과 추천의 기반이 되는 가장 기본적인 기능입니다.

### 7.2 AI 경험 분석

AI는 저장된 특정 활동 경험을 분석하여 나중에 다시 활용하기 쉬운 형태로 정리합니다.

분석 결과에는 다음 내용을 포함합니다.

- 경험 요약
- 핵심 역량 태그
- 주요 성과
- 활용 가능한 키워드
- 분석 생성일

이 기능은 단순히 경험을 저장하는 것을 넘어, 사용자가 자신의 경험에서 어떤 강점과 성과를 말할 수 있는지 이해하도록 돕습니다.

분석 결과는 특정 활동 경험에 연결된 분석 기록으로 localStorage에 저장합니다. 저장 항목은 경험 요약, 핵심 역량 태그, 주요 성과, 활용 가능한 키워드, 분석 생성일입니다.

역량 태그는 고정 분류가 아니라, 대표 예시를 참고해 AI가 생성하는 분석 결과입니다.

대표 예시는 다음과 같습니다.

- 문제 해결력: 문제를 발견하고 원인을 분석해 해결한 경험
- 협업/커뮤니케이션: 팀원과 역할을 나누고 의견을 조율한 경험
- 실행력/주도성: 목표를 정하고 실제 결과물까지 만들어낸 경험
- 분석력/데이터 활용력: 자료, 피드백, 수치, 사용자 반응 등을 바탕으로 판단한 경험

### 7.3 AI 경험 추천 및 활용

사용자가 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문 등을 입력하면 AI는 저장된 경험 전체와 분석 결과를 참고해 가장 적합한 경험 1개를 추천합니다. 이 기능은 특정 경험 하나의 분석 결과에서 반드시 이어지는 흐름이 아니라, 별도의 AI 경험 추천 및 활용 화면에서 실행하는 흐름입니다.

추천 결과에는 다음 내용을 포함합니다.

- 추천 경험
- 추천 이유
- 관련 역량 태그
- 강조할 성과
- 활용 방향
- 자기소개서 / 포트폴리오 / 면접 답변 참고 문장

CampusLog의 핵심 가치는 AI가 단순히 문장을 생성하는 것이 아니라,
사용자가 잊고 있던 경험을 찾아주고 현재 지원 목적에 맞는 경험을 선택하도록 돕는 것입니다.

1차 MVP에서는 완성도 높은 최종 문서를 자동으로 만들어주는 것보다, 사용자가 바로 참고하고 수정할 수 있는 경험 추천과 활용 방향 제공에 집중합니다.

추천 결과 저장은 1차 MVP 범위에 포함하지만, 첫 구현에서는 참고 문장 복사보다 낮은 2순위로 다룹니다.

## 8. MVP에서 제외하는 기능

아래 제외 기능은 1차 MVP 기준입니다. 1차 MVP에서는 범위를 넓히지 않기 위해 아래 기능을 구현하지 않습니다.

- 로그인 / 회원가입
- 학교 인증
- GitHub 자동 연동
- 블로그 자동 연동
- Notion 연동
- 파일 업로드
- 수료증 / 활동사진 첨부
- PDF 포트폴리오 자동 생성
- 관리자 페이지
- 결제
- 모바일 앱
- 팀 협업 기능
- 공개 프로필
- 복잡한 추천 알고리즘
- 여러 추천 후보 탐색
- Supabase Auth / Postgres / Storage 도입
- Spring Boot 백엔드 구현
- MySQL 또는 AWS RDS 구축
- AWS S3 파일 저장

Supabase Auth / Postgres / Storage는 2차 MVP 확장 예정 범위입니다. Spring Boot, MySQL 또는 AWS RDS, AWS S3는 UNIKER 이후 3차 확장 범위입니다. 위 기능들은 1차 MVP 구현 범위에 포함하지 않습니다.

## 9. 사용자 흐름

1차 MVP에서 사용자는 경험 기록 / 분석 흐름과 활용 추천 흐름을 별도 진입 흐름으로 사용합니다.

경험 기록 / 분석 흐름:

1. 메인 화면에서 활동 경험 목록 또는 새 활동 경험 작성으로 진입합니다.
2. 활동 경험 상세 화면에서 원본 내용을 확인합니다.
3. 활동 경험 상세 화면에서 AI 분석 요청을 실행합니다.
4. AI가 해당 경험의 경험 요약, 핵심 역량 태그, 주요 성과, 활용 가능한 키워드, 분석 생성일을 분석 결과로 저장합니다.

활용 추천 흐름:

1. 메인 화면에서 AI 경험 추천 및 활용 화면으로 진입합니다.
2. 사용자가 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문 등을 입력합니다.
3. AI가 저장된 경험 전체와 분석 결과를 참고해 가장 적합한 경험 1개를 추천합니다.
4. 사용자는 추천 이유와 참고 문장을 확인하고, 참고 문장을 복사하거나 추천 결과를 저장해 자신의 지원서나 면접 준비에 활용합니다.

## 10. 화면 구성 초안

1차 MVP에서는 필요한 화면을 최소한으로 구성합니다.

- 메인 화면
- 활동 경험 목록 화면
- 활동 경험 작성 / 수정 화면
- 활동 경험 상세 화면
- AI 경험 분석 결과 화면
- AI 경험 추천 및 활용 화면

## 11. 성공 기준

CampusLog 1차 MVP는 사용자가 다음 일을 할 수 있으면 1차 완성으로 봅니다.

- 사용자가 자신의 활동 경험을 등록할 수 있는가
- 사용자가 등록한 경험을 브라우저 localStorage 기반으로 다시 확인할 수 있는가
- AI가 특정 활동 경험에 연결된 분석 결과를 요약, 핵심 역량 태그, 주요 성과, 키워드로 정리하고 localStorage에 저장할 수 있는가
- 사용자가 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문 등을 입력할 수 있는가
- AI가 저장된 경험 전체와 분석 결과를 참고해 입력된 상황에 맞는 가장 적합한 경험 1개를 추천할 수 있는가
- AI가 추천한 이유를 함께 제공하는가
- 사용자가 추천 결과를 활용해 자기소개서, 포트폴리오, 면접 답변 작성에 참고할 수 있는가
- 경험 기록 / 분석 흐름과 활용 추천 흐름을 로그인, DB, Spring Boot 없이 각각 검증할 수 있는가
