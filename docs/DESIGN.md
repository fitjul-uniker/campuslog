# CampusLog Design Guide

## 1. 디자인 목표

CampusLog 1차 MVP의 디자인 목표는 대학생 사용자가 자신의 활동 경험을 빠르게 찾고, 새 경험을 기록하고, AI 분석과 추천 기능으로 자연스럽게 이어질 수 있는 대시보드형 UI를 만드는 것입니다.

1차 MVP의 첫 화면은 별도 허브형 메인 화면이나 서비스 소개 랜딩이 아니라, 저장된 활동 경험 목록이 바로 보이는 **메인/경험 목록 대시보드**입니다.

핵심 목표는 다음과 같습니다.

- 사용자가 서비스에 들어오면 바로 "내가 기록한 경험"을 볼 수 있어야 합니다.
- 새 경험 기록과 기존 경험 확인이 첫 화면에서 가장 빠르게 가능해야 합니다.
- AI 경험 추천 및 활용 기능은 숨겨진 보조 기능이 아니라, 메인 대시보드에서 자연스럽게 진입 가능한 주요 기능이어야 합니다.
- 1차 MVP에서는 화려한 시각 효과보다 명확한 정보 구조, 안정적인 상태 표현, 예측 가능한 이동 흐름을 우선합니다.
- Codex가 이후 실제 UI를 구현할 때 shadcn/ui 중심으로 빠르게 재현할 수 있는 구조를 기준으로 합니다.

## 2. 디자인 원칙

### 2.1 정보 우선순위

메인/경험 목록 대시보드의 정보 우선순위는 다음 순서로 둡니다.

1. 브랜드명, 메인 화면 제목, 짧은 설명
2. 새 경험 기록하기 버튼
3. 저장된 활동 경험 카드 목록
4. AI 경험 추천 및 활용 진입 버튼
5. 정렬 / 필터
6. 분석 상태 배지와 AI 생성 역량 태그 / 키워드

1차 MVP에서 가장 중요한 행동은 `새 경험 기록하기`입니다. `AI 경험 추천 및 활용`은 메인에서 보이게 하되, 새 경험 기록보다 한 단계 낮은 Secondary CTA로 다룹니다. 저장된 경험이 없는 빈 상태에서는 AI 추천보다 `첫 경험 기록하기`를 먼저 안내합니다.

활동 경험 카드에서는 사용자가 빠르게 스캔할 수 있도록 제목, 기간, 역할 또는 한 줄 설명, 분석 상태, 핵심 태그, 최근 수정일을 우선 노출합니다. 긴 내용과 성과 상세, 생성일은 활동 경험 상세 화면에서 확인합니다.

### 2.2 화면 성격

- 대시보드: 저장된 경험을 탐색하고 주요 작업으로 진입하는 화면입니다.
- 작성 / 수정: 경험 원본 데이터를 입력하고 저장하는 폼 화면입니다.
- 상세: 저장된 경험의 원본 내용 확인, 수정, 삭제, AI 분석 요청을 실행하는 화면입니다.
- AI 분석 결과: 특정 경험에 연결된 분석 결과를 확인하는 화면입니다.
- AI 경험 추천 및 활용: 저장된 경험 전체와 분석 결과를 바탕으로 활용 목적에 맞는 경험 1개를 추천받는 화면입니다.

### 2.3 구현 원칙

- shadcn/ui로 구현 가능한 컴포넌트를 우선 사용합니다.
- 카드, 버튼, 입력, 배지, 탭, 셀렉트, 스켈레톤, 토스트, 다이얼로그는 shadcn/ui 기본 패턴을 따릅니다.
- 아이콘은 가능한 경우 Lucide Icons를 사용합니다.
- Magic UI, Aceternity UI, Motion 효과는 1차 MVP 필수 요소가 아니며 future enhancement로만 검토합니다.
- 카드 안에 또 다른 카드가 들어가는 중첩 카드 구조는 피합니다.
- 큰 랜딩 히어로, 마케팅 카피 중심 섹션, 프로필 메뉴, 계정 메뉴는 1차 MVP에 만들지 않습니다.
- 1차 MVP는 localStorage 기반이므로 로그인 사용자 정보, 조직, 팀, 권한 상태를 전제로 한 UI를 만들지 않습니다.

### 2.4 디자인 토큰 기준

아래 기준은 확정 hex 값이 아니라 구현 시 역할을 맞추기 위한 토큰 기준입니다. 1차 MVP에서는 정확한 hex 값을 문서에서 고정하지 않고, 색상의 역할과 대비 기준을 우선합니다.

- 배경색: 전체 앱 배경은 순백보다 약간 부드러운 neutral 또는 그린 / 민트 계열의 아주 옅은 톤을 사용합니다.
- 메인 컬러: CampusLog의 메인 컬러는 캠퍼스 그린으로 둡니다.
- 보조 컬러: 민트 계열을 보조 컬러로 사용해 편안하게 기록하고 싶은 느낌을 만듭니다.
- 제한적 보조 색상: 인디고 블루는 신뢰감을 주는 보조 색상 후보로만 제한적으로 검토합니다.
- 메인 콘텐츠 카드 배경: 경험 카드와 폼 영역은 읽기 쉬운 흰색 또는 near-white 계열을 우선합니다.
- 텍스트 색상 역할: 제목 / 본문 / 보조 설명 / 비활성 / 오류 텍스트 역할을 구분하고, 본문 가독성을 최우선으로 둡니다.
- border 기준: 카드, 입력, 분리선에는 옅은 neutral border를 사용하고, focus 상태는 캠퍼스 그린 또는 민트 계열을 활용할 수 있습니다.
- 접근성 기준: 텍스트와 버튼의 대비가 낮아지지 않도록 색감보다 가독성과 명확한 상태 구분을 우선합니다.
- radius 기준: 카드, 버튼, 입력의 radius는 8px 이하를 기준으로 하며, 과하게 둥근 pill 형태는 배지나 짧은 태그에만 제한적으로 사용합니다.
- spacing 기준: shadcn/ui 기본 간격과 4px 단위 스케일을 기준으로 화면 안의 여백 리듬을 맞춥니다.
- card padding: 데스크톱 카드 내부는 20~24px 수준, 모바일은 16px 수준을 기준으로 둡니다.
- page padding: 데스크톱은 24~32px 수준, 모바일은 16px 수준을 기준으로 둡니다.
- shadow 사용 기준: 기본 카드에는 border 중심의 낮은 대비를 사용하고, shadow는 hover, dialog, popover처럼 떠 있는 계층을 표현할 때만 약하게 사용합니다.

## 3. 참고 레퍼런스

첨부된 Slido dashboard 이미지는 다음 구조만 참고합니다.

- 좌측 사이드바 또는 주요 내비게이션
- 상단 검색 또는 액션 영역
- 중앙의 리스트형 카드 목록
- 우측 상단의 Primary CTA 버튼
- 넓은 여백과 정돈된 대시보드형 화면 구성

CampusLog에서는 이를 아래처럼 해석합니다.

| Slido 레퍼런스 구조 | CampusLog 해석 |
|---|---|
| My slidos | 나의 경험 / 경험 목록 |
| Create slido | 새 경험 기록하기 |
| 이벤트 리스트 | 활동 경험 카드 목록 |
| Active and upcoming | 나의 활동 경험 |
| 좌측 메뉴 | 나의 경험, AI 추천 및 활용 정도의 최소 내비게이션 |
| 상단 검색 영역 | 대시보드 상단 액션 영역. 1차 MVP에서는 정렬 / 필터 중심으로 사용하고, 작동하지 않는 장식용 검색창은 만들지 않음 |
| 프로필 / 팀 / 업그레이드 / 외부 연동 | 1차 MVP 제외, future scope |

Slido의 브랜드, 색상, 문구, 메뉴 이름, 조직 / 팀 / 업그레이드 / 연동 구조는 복제하지 않습니다.

## 4. 브랜드 / 비주얼 방향

이 섹션은 팀 피드백 반영 후 1차 MVP 기준으로 정리한 브랜드 / 비주얼 방향입니다. 기존 회의 메모의 후보 아이디어 중 대표 컨셉은 확정하고, 보조 모티프와 future scope는 분리합니다.

### 4.1 대표 컨셉

#### 노트 / 단권화 컨셉

CampusLog의 대표 컨셉은 **노트 / 단권화**입니다.

핵심 메시지는 다음으로 정리합니다.

- 대학생활을 단권화하는 AI 경험 기록장
- 대학생활을 단권화하다
- 흩어진 경험을 기록하고, 필요할 때 AI로 다시 찾아보세요.

사용자의 대학교 4~6년 경험을 하나의 웹사이트 노트로 단권화한다는 이미지입니다. AI는 책갈피처럼 필요한 경험을 빠르게 찾아주는 도우미로 해석합니다.

CampusLog의 `기록하다`, `나중에 다시 꺼내 쓰다`, `경험을 정리하다`라는 핵심 가치와 잘 맞습니다. 1차 MVP의 기록 / 목록 / 상세 / 분석 흐름에서도 가장 자연스럽게 연결됩니다.

### 4.2 보조 모티프 / 보류 컨셉

#### 레고 / 테트리스 컨셉

레고 / 테트리스 컨셉은 메인 컨셉에서 내리고, 태그 / 배지 / 카드 조각 느낌 같은 보조 모티프로만 남깁니다.

사용자의 활동 경험, 역량, 성과, 키워드를 각각의 조각으로 보고, 이를 조립해 자기소개서, 포트폴리오, 면접 답변 같은 결과물로 만들어낸다는 해석은 AI 추천 / 활용 기능과 연결할 수 있습니다. 다만 1차 MVP에서는 장난감처럼 보이거나 과한 그래픽으로 흐르지 않도록, 카드 구성과 태그 묶음 정도에서만 은근히 반영합니다.

#### 정육면체 / 모듈 컨셉

하나의 경험을 여러 면을 가진 정육면체처럼 보고, 제목 / 기간 / 역할 / 성과 / 역량 / 키워드가 각각의 면으로 정리된다는 이미지입니다.

경험 하나가 여러 관점에서 해석되고, AI가 그 면들을 정리해주는 느낌을 줄 수 있습니다.

정육면체 / 모듈 컨셉은 1차 MVP에서는 보류합니다. 추후 브랜드 고도화 단계에서 검토하되, 3D 그래픽이나 복잡한 애니메이션으로 구현하지 않는 것을 기본 원칙으로 둡니다.

### 4.3 컬러 방향

1차 MVP의 컬러 방향은 **캠퍼스 그린 + 민트**입니다.

- 메인 컬러: 캠퍼스 그린
- 보조 컬러: 민트 계열
- 제한적 보조 색상 후보: 인디고 블루

전체 분위기는 `푸릇푸릇한 봄의 캠퍼스`, `편안하게 기록하고 싶은 느낌`, `대학생의 성장과 활동`으로 정리합니다.

정확한 hex 값은 아직 확정하지 않고 역할 중심으로 작성합니다. 너무 밝은 연두만 사용하면 가벼워 보일 수 있으므로, 메인 액션과 focus 상태에는 충분한 대비를 가진 그린을 사용하고, 민트는 배경 톤, hover, 보조 포인트에 가볍게 사용합니다.

인디고 블루는 서비스의 신뢰감을 보완하는 색상 후보로만 제한적으로 남깁니다. 메인 컬러처럼 넓게 쓰지 않고, 필요한 경우 링크, 정보성 상태, 차분한 보조 강조 정도로 검토합니다.

실제 구현에서는 접근성 대비를 해치지 않도록 텍스트와 버튼 대비를 우선합니다.

### 4.4 파비콘 / 로고 방향

#### 책갈피가 꽂힌 노트

1차 MVP의 파비콘 후보는 **책갈피가 꽂힌 노트**입니다.

책갈피 노트 아이콘은 `기록`과 `다시 찾아보기`를 함께 표현합니다. 노트는 대학생활 경험을 차곡차곡 기록하는 성격을, 책갈피는 AI가 필요한 경험을 빠르게 찾아주는 도우미라는 의미를 담습니다.

#### 단순 노트 아이콘

단순 노트 아이콘은 보조 후보로 남깁니다. CampusLog의 `기록` 정체성과 직접적으로 연결되어 안전하지만, 책갈피형 노트보다 `다시 찾아보기`의 의미는 약합니다.

#### 레고 / 테트리스 조각

- 경험과 역량을 조립한다는 컨셉을 표현하기 좋습니다.
- 1차 MVP의 파비콘 / 로고 메인 후보에서는 제외하고, future scope 또는 보조 그래픽으로 검토합니다.

#### 정육면체 / 블록

- 경험이 여러 관점으로 정리된다는 느낌을 줄 수 있습니다.
- 1차 MVP에서는 보류합니다.

### 4.5 메인 화면 비주얼 방향

- 메인 / 경험 목록 대시보드는 기능 중심의 실용적인 화면이므로 과한 장식보다 여백, 카드 정렬, 상태 배지, CTA 배치가 중요합니다.
- 브랜드 감성은 캠퍼스 그린 / 민트 포인트 컬러, 책갈피 노트 파비콘, 빈 상태 아이콘, 상태 배지 정도로 표현합니다.
- 카드 상단의 작은 포인트 바, 상태 배지의 색상 포인트, 부드러운 hover transition은 선택적으로 사용할 수 있습니다.
- `기능 조각이 모여 하나의 결과물이 된다`는 느낌은 카드, 배지, 태그의 조합으로만 가볍게 표현합니다.
- 1차 MVP에서는 3D 그래픽, 복잡한 일러스트, 마스코트, 과한 애니메이션, 랜딩페이지용 대형 히어로 이미지는 사용하지 않습니다.

### 4.6 브랜드 그래픽 / 모션 범위

1차 MVP의 브랜드 그래픽 범위는 최소~기본 수준으로 둡니다.

필수:

- CampusLog 텍스트 로고
- 책갈피 노트 파비콘
- 빈 상태 아이콘
- 그린 / 민트 포인트 컬러

선택:

- 카드 상단 포인트 바
- 상태 배지의 색상 포인트
- 부드러운 hover transition
- 페이지 전환 또는 버튼 interaction의 아주 약한 motion

제외:

- 3D 그래픽
- 복잡한 일러스트
- 마스코트
- 과한 애니메이션
- 랜딩페이지용 대형 히어로 이미지

`애플 느낌처럼 자연스럽고 과하지 않은 움직임`은 1차 MVP에서는 hover, transition, loading feedback 정도로만 제한합니다.

### 4.7 팀 피드백 반영 결과

- 대표 컨셉: 노트 / 단권화
- 핵심 메시지: 대학생활을 단권화하는 AI 경험 기록장
- 메인 컬러: 캠퍼스 그린 + 민트
- 파비콘: 책갈피형 노트
- 데스크톱 레이아웃: 좌측 사이드바형
- 모바일 레이아웃: 상단 앱 바 + 세로 스크롤 + 하단 / 아이콘 중심 액션
- 브랜드 그래픽: 1차 MVP에서는 최소~기본 수준

아직 미정인 세부 값은 추후 구현 후 화면을 보며 조정합니다. 이번 프로젝트에서는 Figma를 필수 작업으로 두지 않고, Pageflows / Slido 레퍼런스와 이 DESIGN.md를 기준으로 Codex에게 바로 구현을 맡기는 흐름을 기준으로 합니다.

## 5. 전체 레이아웃 구조

### 5.1 데스크톱 기본 구조

데스크톱 1차 구현 기준은 **좌측 사이드바형** 앱 셸입니다. 좌측 사이드바는 Slido 레퍼런스의 구조만 참고하되, 무겁게 만들지 않고 미니멀하게 유지합니다.

상단 내비게이션형은 1차 MVP에서는 보류하고, 추후 디자인 고도화 시 재검토하는 옵션으로 남깁니다.

```text
1차 MVP 기준. 좌측 사이드바형
┌────────────────────────────────────────────────────────────┐
│ 미니멀 좌측 사이드바 │ 상단 액션 영역                         │
│               ├────────────────────────────────────────────┤
│               │ 메인 콘텐츠                                │
│               │ - 나의 CampusLog / 짧은 설명               │
│               │ - 나의 활동 경험 섹션                      │
│               │ - 정렬 / 필터 / 주요 CTA                   │
│               │ - 리스트형 경험 카드                       │
│               │ - 빈 상태 / 오류 상태                      │
└────────────────────────────────────────────────────────────┘

보류. 상단 내비게이션형
┌────────────────────────────────────────────────────────────┐
│ 상단 앱 바 / 주요 내비게이션 / 주요 CTA                    │
├────────────────────────────────────────────────────────────┤
│ 메인 콘텐츠                                                │
│ - 화면 제목 / 짧은 설명                                    │
│ - 정렬 / 필터 / 주요 CTA                                   │
│ - 리스트형 경험 카드                                       │
│ - 빈 상태 / 오류 상태                                      │
└────────────────────────────────────────────────────────────┘
```

### 5.2 내비게이션 구조

좌측 사이드바는 1차 MVP 기준의 데스크톱 내비게이션입니다. 메뉴 수가 적으므로 폭, 색상, 그림자, 장식을 과하게 주지 않고 미니멀하게 구성합니다.

1차 MVP에서는 최소 메뉴만 제공합니다.

- CampusLog
- 나의 경험
- AI 추천 및 활용

포함하지 않을 항목:

- 로그인 / 회원가입
- 프로필
- 마이페이지
- 팀 / 조직 관리
- 업그레이드 / 결제
- 외부 서비스 연동
- 관리자 메뉴

권장 Lucide Icons:

- CampusLog 또는 나의 경험: `BookOpen`, `NotebookText`
- AI 추천 및 활용: `Sparkles`
- 새 경험 기록하기: `SquarePen`, `PencilLine`
- 보조 추가 액션: `Plus`
- 대시보드로 돌아가기: `ArrowLeft`

### 5.3 상단 액션 영역

상단 액션 영역은 현재 화면의 주요 작업을 빠르게 실행하는 영역입니다.

메인/경험 목록 대시보드에서는 다음을 둡니다.

- 브랜드명: CampusLog
- 메인 화면 제목: 나의 CampusLog
- 목록 섹션 제목: 나의 활동 경험
- 짧은 설명: 흩어진 경험을 기록하고, 필요할 때 AI로 다시 찾아보세요.
- 정렬 옵션
- 필터 옵션
- AI 경험 추천 및 활용 버튼
- 새 경험 기록하기 Primary CTA

대시보드에는 긴 설명 문구를 많이 넣지 않고 짧고 담백한 문구만 사용합니다. `서비스 이름과 짧은 설명`이 설명문처럼 무겁게 보이지 않도록 실제 구현에서는 브랜드 영역과 메인 제목 주변에 시각적으로 가볍게 배치합니다.

검색 입력은 1차 MVP 필수 구현 범위가 아닙니다. 첫 구현에서는 정렬 / 필터를 우선하고, 검색은 구현 여유가 있을 때 localStorage에 저장된 경험 제목, 역할, 키워드 기준의 클라이언트 검색으로 확장합니다. 작동하지 않는 장식용 검색창은 만들지 않습니다.

### 5.4 중앙 리스트 카드

경험 목록은 리스트형 카드를 기본으로 합니다.

- 전체 너비를 과하게 쪼개지 않고, 한 행에 하나의 경험을 보여줍니다.
- 카드 높이는 내용량에 따라 크게 흔들리지 않게 기본 정보를 중심으로 고정감 있게 구성합니다.
- 카드 클릭 시 활동 경험 상세 화면으로 이동합니다.
- 카드 내부의 수정 / 삭제 같은 세부 액션은 1차 MVP에서는 상세 화면에서 처리하는 것을 우선합니다.

### 5.5 우측 상단 CTA

데스크톱에서는 `새 경험 기록하기`를 우측 상단 Primary CTA로 둡니다.

`AI 경험 추천 및 활용`은 같은 상단 액션 영역에 Secondary CTA 또는 강조된 보조 버튼으로 둡니다. AI 기능이 숨겨지지 않도록 내비게이션 메뉴와 상단 액션 중 하나 이상에서 접근 가능해야 하지만, 새 경험 기록하기보다 시각적 우선순위가 높아지면 안 됩니다.

저장된 경험이 없는 경우에는 `AI 경험 추천 및 활용`보다 `첫 경험 기록하기`를 우선 안내합니다. 다만 AI 추천 진입을 완전히 숨기지는 않고, 비활성 / 흐린 처리 / 낮은 우선순위 버튼 중 하나로 보여줄 수 있습니다.

## 6. 화면별 예상 라우트

아래 라우트는 1차 MVP 화면 구조를 정리하기 위한 예상안입니다. 라우트 추가가 로그인 / 회원가입, 마이페이지, 파일 첨부, 여러 추천 후보 비교 같은 제외 범위를 새로 포함한다는 의미는 아닙니다.

## 7. 화면별 디자인 기준

### 7.1 메인/경험 목록 대시보드

### 목적

사용자가 CampusLog에 들어왔을 때 저장된 활동 경험 목록을 바로 보고, 새 경험 기록, 기존 경험 상세 확인, AI 경험 추천 및 활용 화면으로 이동할 수 있게 합니다.

### 레이아웃

- 데스크톱: 미니멀 좌측 사이드바형 앱 셸 + 상단 액션 영역 + 중앙 리스트 카드
- 태블릿: 사이드바 폭 축소 또는 아이콘 중심 내비게이션
- 모바일: 상단 앱 바 + 세로 스크롤 + 하단 / 아이콘 중심 액션

### 주요 구성

- 브랜드명: CampusLog
- 메인 화면 제목: 나의 CampusLog
- 짧은 설명: 흩어진 경험을 기록하고, 필요할 때 AI로 다시 찾아보세요.
- 목록 섹션 제목: 나의 활동 경험
- 저장된 활동 경험 카드 목록
- 새 경험 기록하기 버튼
- AI 경험 추천 및 활용 버튼
- 정렬 옵션
- 필터 옵션
- 분석 상태 배지
- AI 생성 역량 태그 / 키워드
- 빈 상태 CTA: 첫 경험 기록하기

1차 MVP에는 로그인 기능이 없으므로 실제 사용자 이름 기반 문구는 사용하지 않습니다. 개인 이름이 들어가는 제목 대신 기본값은 `나의 CampusLog`로 둡니다.

빈 상태에서는 `첫 경험 기록하기`를 가장 우선 안내합니다. `AI 경험 추천 및 활용`은 완전히 숨기지 않고, 비활성 / 흐린 처리 / 낮은 우선순위 버튼으로 보여줄 수 있습니다. 사용자가 누르거나 접근하면 `경험을 먼저 기록하면 AI 추천을 받을 수 있어요`처럼 먼저 기록이 필요하다는 안내를 보여줍니다.

### 경험 카드 정보 구조

경험 카드에는 다음 순서로 정보를 배치합니다.

1. 제목
2. 기간
3. 역할 또는 한 줄 설명
4. 분석 상태 배지
5. AI 생성 역량 태그 또는 키워드
6. 최근 수정일
7. 관련 링크 존재 여부

카드 전체는 클릭 가능한 영역으로 만들되, 버튼처럼 과도하게 보이지 않게 리스트 아이템 패턴을 사용합니다. 키보드 접근 시 focus ring이 보여야 합니다.

날짜 정보는 카드에서 모두 크게 노출하지 않습니다.

- 기본 노출: 최근 수정일
- 보조 메타 또는 상세 화면: 생성일
- 정렬 기준: 최근 수정순, 오래된 작성순, 활동 기간순

### 정렬 / 필터

정렬 옵션:

- 최근 수정순
- 오래된 작성순
- 활동 기간순

필터 옵션:

- 전체
- 미분석
- 분석 완료
- 재분석 필요
- 역량 태그 / 키워드 기준 필터

권장 shadcn/ui:

- `Select`: 정렬 옵션
- `DropdownMenu`: 필터 옵션이 여러 개일 때
- `Badge`: 분석 상태와 태그
- `Button`: 새 경험 기록하기, AI 경험 추천 및 활용
- `Card` 또는 리스트 스타일 `div`: 경험 카드
- `Skeleton`: 목록 로딩
- `Alert`: 오류 상태

권장 Lucide Icons:

- `SquarePen` 또는 `PencilLine`: 새 경험 기록하기
- `Plus`: 보조 추가 아이콘
- `Sparkles`: AI 경험 추천 및 활용
- `Filter`: 필터
- `ArrowUpDown`: 정렬
- `CheckCircle2`: 분석 완료

### 7.2 활동 경험 작성 / 수정 화면

### 목적

사용자가 활동 경험 원본 데이터를 입력하거나 수정하는 화면입니다. AI 분석 요청은 이 화면에서 실행하지 않습니다.

### 레이아웃

- 중앙 정렬된 단일 폼 레이아웃을 사용합니다.
- 폼 최대 너비는 데스크톱 기준 720~840px 안팎으로 둡니다.
- 저장 / 취소 버튼은 폼 하단에 고정된 액션 영역처럼 배치합니다.

### 필드

- 제목
- 기간
- 역할
- 내용
- 성과
- 관련 링크

### UI 기준

- 필수 입력값은 label과 validation message로 명확히 표시합니다.
- 긴 텍스트 입력은 `Textarea`를 사용합니다.
- 기간은 단순 텍스트 또는 날짜 입력 중 구현 난이도에 맞게 선택하되, 1차 MVP에서는 빠른 입력을 우선합니다.
- 파일 첨부 UI는 포함하지 않습니다.

권장 shadcn/ui:

- `Input`
- `Textarea`
- `Button`
- `Label`
- `Form` 또는 기존 폼 패턴

권장 Lucide Icons:

- `Link`: 관련 링크

### 7.3 활동 경험 상세 화면

### 목적

저장된 활동 경험의 원본 내용을 확인하고, 수정하거나 AI 분석을 요청할 수 있게 합니다.

### 정보 우선순위

1. 제목
2. 기간 / 역할
3. 내용
4. 성과
5. 관련 링크
6. 생성일 / 수정일
7. AI 분석 상태
8. 핵심 태그 일부
9. 분석 결과 preview
10. 수정 / AI 분석 요청 / AI 분석 결과 보기

### UI 기준

- 원본 경험 내용과 AI 분석 상태를 같은 화면에서 확인할 수 있어야 합니다.
- 분석 결과 전체를 상세 화면에서 모두 펼쳐 보여주지 않습니다.
- 상세 화면에는 AI 분석 상태, 핵심 태그 일부, 분석 결과 preview 정도만 보여줍니다.
- 전체 분석 결과는 별도 `AI 경험 분석 결과 화면`에서 확인합니다.
- `AI 분석 요청`은 명확한 Primary 또는 강조 버튼으로 제공합니다.
- 분석 결과가 있으면 `AI 분석 결과 보기` 버튼을 제공합니다.
- 상세 화면이 길어지는 것을 막고, `/experiences/[id]/analysis` 라우트 구조를 유지하기 위해 분석 결과 화면을 분리합니다.
- 수정은 Secondary 액션으로 둡니다.
- 삭제가 구현되는 경우 destructive variant와 확인 다이얼로그를 사용합니다.

- AI 추천으로 이동하는 버튼은 Secondary CTA 또는 보조 버튼으로 둡니다.

- `RefreshCw`: 다시 분석
- `Calendar`: 분석 생성일

### 7.5 AI 경험 추천 및 활용 화면

### 목적

사용자가 자기소개서 문항, 포트폴리오 목적, 면접 질문, 기타 활용 목적을 입력하면 저장된 경험 전체와 분석 결과를 참고해 가장 적합한 경험 1개를 추천합니다.

### 레이아웃

- 상단: 활용 목적 선택과 질문 / 문항 입력
- 중단: 추천 요청 버튼
- 하단: 추천 경험 카드 1개와 추천 이유, 활용 방향, 참고 문장

### UI 기준

- 1차 MVP에서는 추천 경험을 1개만 보여줍니다.
- 여러 후보를 비교하는 UI는 만들지 않습니다.
- 복사 기능을 우선합니다.
- 추천 결과 저장은 MVP에 포함하지만, 첫 구현에서는 복사보다 낮은 우선순위로 둡니다.
- 저장된 경험이 없는 경우 먼저 경험을 기록하도록 안내합니다.

권장 shadcn/ui:

- `Select` 또는 `Tabs`: 활용 목적 선택
- `Textarea`: 질문 / 문항 입력
- `Button`: 추천 요청, 복사, 저장
- `Card`: 추천 경험 결과
- `Alert`: 입력 없음 / 경험 없음 / 실패 상태
- `Skeleton`: 추천 요청 중

권장 Lucide Icons:

- `Sparkles`: 추천 요청
- `Copy`: 참고 문장 복사
- `Save`: 추천 결과 저장
- `FileQuestion`: 질문 / 문항
- `ArrowLeft`: 대시보드로 돌아가기

## 8. 컴포넌트 기준

### 8.1 Button

- Primary: 새 경험 기록하기, 저장, AI 분석 요청, 추천 요청
- Secondary: AI 경험 추천 및 활용, 취소, 대시보드로 돌아가기
- Ghost: 필터 열기, 보조 이동
- Destructive: 삭제

메인 / 경험 목록 대시보드에서는 `새 경험 기록하기`가 가장 중요한 Primary CTA입니다. `AI 경험 추천 및 활용`은 보이게 유지하되 Secondary CTA로 두고, 빈 상태에서는 `첫 경험 기록하기`보다 앞서지 않게 합니다.

버튼 안에는 가능한 경우 Lucide 아이콘을 함께 사용합니다. 단, 모든 버튼에 아이콘을 강제로 넣지 않고, 주요 액션과 의미가 명확한 액션에만 사용합니다.

`새 경험 기록하기`는 기록 서비스의 성격상 단순 추가를 의미하는 `Plus`보다 작성 행위를 떠올리기 쉬운 `SquarePen` 또는 `PencilLine` 계열 아이콘을 우선 추천합니다. `Plus`는 보조 추가 아이콘으로 남깁니다.

### 8.2 Badge

분석 상태 배지:

- 미분석: 회색 계열
- 분석 완료: 녹색 또는 청록 계열
- 재분석 필요: 노란색 또는 주황 계열

역량 태그 / 키워드:

- 정보 태그로 표현하되 분석 상태 배지와 색이 섞이지 않게 구분합니다.
- 한 카드에서 태그가 너무 많으면 2~3개까지만 노출하고 나머지는 `+N`으로 축약합니다.

### 8.3 Card / List Item

- 메인 대시보드의 경험 목록은 리스트형 카드로 구현합니다.
- 카드 radius는 8px 이하를 기준으로 합니다.
- 카드 내부에는 제목, 메타 정보, 상태, 태그가 한눈에 들어오게 배치합니다.
- 카드 클릭 영역과 내부 액션이 충돌하지 않게 합니다.

### 8.4 Form

- label은 항상 노출합니다.
- placeholder는 예시를 보조하는 용도로만 사용합니다.
- validation message는 입력 필드 바로 아래에 표시합니다.
- 필수 입력 누락 시 저장하지 않고 안내합니다.

### 8.5 Feedback

- 일시적 성공 안내는 `Toast`를 사용할 수 있습니다.
- 데이터 로딩 실패, AI 요청 실패처럼 사용자가 조치를 해야 하는 상태는 `Alert`로 화면 안에 표시합니다.
- 삭제 확인은 `AlertDialog`를 사용합니다.

### 9.1 저장된 경험이 없는 빈 상태

표시 위치:

- 메인/경험 목록 대시보드의 목록 영역

구성:

- 제목: 아직 기록한 활동 경험이 없습니다

- Primary CTA: 첫 경험 기록하기
- 낮은 우선순위 CTA: AI 경험 추천 및 활용

## 10. 모바일 반응형 기준

### 10.1 브레이크포인트 기준

- 모바일: 0~767px
- 태블릿: 768~1023px
- 데스크톱: 1024px 이상

### 10.2 모바일 대시보드

- 좌측 사이드바는 노출하지 않습니다.
- 모바일은 상단 앱 바 + 세로 스크롤 중심 레이아웃으로 정리합니다.
- 상단 앱 바에는 CampusLog 이름과 필요한 최소 액션만 둡니다.
- 주요 액션은 화면 상단 또는 하단 고정 액션 영역에서 빠르게 접근할 수 있게 합니다.
- `새 경험 기록하기`는 Primary CTA로 유지합니다.
- `AI 경험 추천 및 활용`은 Primary CTA 아래 또는 옆에 Secondary CTA로 배치합니다.
- `새 경험 기록하기`와 `AI 경험 추천 및 활용`은 모바일에서 짧은 문구 또는 아이콘 중심으로 표현합니다.
- 예: 새 기록
- 예: AI 추천
- 하단 고정 액션 영역 또는 화면 하단 플로팅 액션을 검토할 수 있습니다.
- 단, 1차 MVP에서는 사용성을 해치지 않는 단순한 구조를 우선합니다.
- 정렬 / 필터는 한 줄에 배치하되, 좁으면 필터 버튼 하나로 접습니다.
- 경험 카드는 한 열 리스트로 표시합니다.

### 10.3 모바일 폼

- 입력 필드는 한 열로 배치합니다.
- 저장 / 취소 버튼은 하단에 나란히 두거나, 저장 버튼을 전체 너비로 둡니다.
- 긴 textarea는 충분한 높이를 주되 화면을 과도하게 차지하지 않게 합니다.

### 10.4 모바일 AI 결과 / 추천

- 분석 결과 섹션은 세로 스택으로 배치합니다.
- 태그는 줄바꿈 가능한 wrap 구조를 사용합니다.
- 참고 문장 영역은 복사 버튼이 항상 가까이에 있어야 합니다.

## 12. 1차 MVP에서 제외할 디자인 요소

1차 MVP에서는 아래 UI를 만들지 않습니다.

- 로그인 / 회원가입 화면
- 서비스 소개 전용 랜딩 페이지
- 마이페이지
- 프로필 메뉴
- 사용자 아바타 / 계정 드롭다운
- 팀 / 조직 관리 메뉴
- 업그레이드 / 결제 UI
- 관리자 화면
- 공개 프로필 화면
- 파일 첨부 UI
- 수료증 / 활동사진 업로드 UI
- PDF 포트폴리오 생성 UI
- GitHub / 블로그 / Notion 자동 연동 설정 UI
- Supabase 사용자 / 파일 관리 UI
- 여러 추천 후보 비교 UI
- 복잡한 차트 / 분석 대시보드
- 과한 애니메이션, 3D, 배경 장식 효과

Magic UI, Aceternity UI, Motion 기반 장식 효과는 1차 MVP 필수 범위가 아닙니다. 이후 사용하더라도 버튼 hover, 화면 전환, 로딩 피드백처럼 사용성을 해치지 않는 작은 향상에 한정합니다.
