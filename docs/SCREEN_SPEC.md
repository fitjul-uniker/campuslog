# CampusLog Screen Spec

## Purpose

이 문서는 CampusLog 1차 MVP의 핵심 화면별 목적, UI 구성 요소, 입력값, 출력값, 버튼 동작, 빈 상태, 오류 상태를 정리합니다.

## 0. 메인 화면

### 목적

사용자가 CampusLog의 핵심 기능으로 진입할 수 있는 시작 화면입니다.

### UI 구성 요소

- 서비스 이름 / 간단한 설명
- 활동 경험 목록으로 이동하는 카드 또는 버튼
- 새 활동 경험 작성으로 이동하는 버튼
- AI 경험 추천 및 활용 화면으로 이동하는 카드 또는 버튼
- 저장된 경험이 없을 때 먼저 활동 경험을 기록하라는 안내

### 입력값

- 없음

### 출력값

- 주요 기능 진입 경로
- 저장된 경험이 없을 때의 안내 메시지

### 버튼 동작

- 활동 경험 목록 보기: 활동 경험 목록 화면으로 이동
- 새 경험 작성: 활동 경험 작성 화면으로 이동
- AI 경험 추천 및 활용 시작하기: AI 경험 추천 및 활용 화면으로 이동

### 빈 상태

- 저장된 경험이 없으면 "먼저 활동 경험을 기록해보세요" 안내와 새 경험 작성 버튼을 보여줍니다.

### 오류 상태

- 저장된 데이터를 불러오지 못하면 다시 시도 안내를 보여줍니다.

## 1. 활동 경험 목록 화면

### 목적

사용자가 저장한 활동 경험을 한눈에 확인하고, 새 경험을 기록하거나 기존 경험 상세로 이동할 수 있게 합니다.

### UI 구성 요소

- 화면 제목
- 활동 경험 카드 목록
- 새 경험 추가 버튼
- 정렬 옵션
- 주요 키워드 / 역량 태그 필터
- 각 경험 카드의 제목
- 각 경험 카드의 기간
- 각 경험 카드의 대표 역할 또는 한 줄 설명
- 각 경험 카드의 주요 키워드 / 역량 태그
- 각 경험 카드의 AI 분석 상태 배지: 분석 완료 / 미분석

### 입력값

- 정렬 기준: 최신순, 오래된순, 활동 기간순
- 필터 기준: 주요 키워드 또는 역량 태그

### 출력값

- 저장된 활동 경험 목록
- 정렬 / 필터가 적용된 활동 경험 목록

### 버튼 동작

- 새 경험 추가: 활동 경험 작성 화면으로 이동
- 경험 카드 선택: 활동 경험 상세 화면으로 이동
- AI 분석 보기는 목록 화면의 주요 액션에서 제외

### 빈 상태

- 저장된 경험이 없으면 "아직 기록한 활동 경험이 없습니다" 메시지와 새 경험 추가 버튼을 보여줍니다.

### 오류 상태

- 저장된 데이터를 불러오지 못하면 다시 시도 안내를 보여줍니다.

## 2. 활동 경험 작성 / 수정 화면

### 목적

사용자가 프로젝트, 공모전, 인턴, 대외활동 등 자신의 활동 경험을 직접 입력하거나 수정할 수 있게 합니다.

### UI 구성 요소

- 제목 입력
- 기간 입력
- 역할 입력
- 내용 입력
- 성과 입력
- 관련 링크 입력
- 저장 버튼
- 취소 버튼

1차 MVP에서는 수료증, 활동사진 등 파일 첨부 UI를 포함하지 않습니다.
작성 / 수정 화면에서는 AI 분석 요청 버튼을 제공하지 않습니다.

### 입력값

- 제목
- 기간
- 역할
- 내용
- 성과
- 관련 링크

### 출력값

- 저장된 활동 경험

### 버튼 동작

- 저장: 입력한 경험을 저장하고 활동 경험 상세 화면으로 이동
- 취소: 입력 내용을 저장하지 않고 이전 화면 또는 활동 경험 목록 화면으로 이동

### 빈 상태

- 새 경험 작성 시 입력 필드는 비어 있는 상태로 보여줍니다.

### 오류 상태

- 필수 입력값이 비어 있으면 저장하지 않고 안내 메시지를 보여줍니다.
- 관련 링크 형식이 올바르지 않으면 링크 확인 안내를 보여줍니다.

## 3. 활동 경험 상세 화면

### 목적

저장된 활동 경험의 원본 내용을 확인하고, 수정하거나 AI 분석을 요청할 수 있게 합니다.

### UI 구성 요소

- 제목
- 기간
- 역할
- 내용
- 성과
- 관련 링크
- 주요 키워드 / 역량 태그
- AI 분석 상태
- 수정 버튼
- AI 분석 요청 버튼
- AI 분석 결과 보기 버튼
- 목록으로 돌아가기 버튼

### 입력값

- 저장된 활동 경험

### 출력값

- 활동 경험 원본 내용
- 주요 키워드 / 역량 태그
- AI 분석 상태

### 버튼 동작

- 수정: 활동 경험 작성 / 수정 화면으로 이동
- AI 분석 요청: 해당 경험을 기준으로 AI 경험 분석 결과 화면으로 이동
- AI 분석 결과 보기: 저장된 AI 경험 분석 결과 화면으로 이동
- 목록으로 돌아가기: 활동 경험 목록 화면으로 이동

### 빈 상태

- 아직 AI 분석 결과가 없으면 "아직 AI 분석 결과가 없습니다" 메시지와 AI 분석 요청 버튼을 보여줍니다.

### 오류 상태

- 저장된 경험을 불러오지 못하면 활동 경험 목록으로 돌아가는 안내를 보여줍니다.

## 4. AI 경험 분석 결과 화면

### 목적

특정 활동 경험에 연결된 AI 분석 결과를 보여줍니다.

### UI 구성 요소

- 경험 요약 영역
- 핵심 역량 태그
- 주요 성과 목록
- 활용 가능한 키워드
- 분석 생성일
- 다시 분석 버튼
- 활동 경험 상세로 돌아가기 버튼
- 활동 경험 목록으로 돌아가기 버튼
- AI 경험 추천 및 활용으로 이동 보조 버튼

### 입력값

- 저장된 활동 경험

### 출력값

- 경험 요약
- 핵심 역량 태그
- 주요 성과
- 활용 가능한 키워드
- 분석 생성일

분석 결과는 특정 활동 경험에 연결된 분석 기록입니다. 분석 결과는 localStorage에 저장되어 사용자가 나중에 다시 확인할 수 있도록 합니다.
역량 태그는 고정 분류가 아니라, 대표 예시를 참고해 AI가 생성하는 분석 결과입니다.
대표 예시는 문제 해결력(문제를 발견하고 원인을 분석해 해결한 경험), 협업/커뮤니케이션(팀원과 역할을 나누고 의견을 조율한 경험), 실행력/주도성(목표를 정하고 실제 결과물까지 만들어낸 경험), 분석력/데이터 활용력(자료, 피드백, 수치, 사용자 반응 등을 바탕으로 판단한 경험)입니다.

### 버튼 동작

- 다시 분석: 같은 경험을 기준으로 AI 분석을 다시 요청
- 활동 경험 상세로 돌아가기: 해당 활동 경험 상세 화면으로 이동하는 Primary CTA
- 활동 경험 목록으로 돌아가기: 활동 경험 목록 화면으로 이동하는 Primary CTA
- AI 경험 추천 및 활용으로 이동: AI 경험 추천 및 활용 화면으로 이동할 수 있는 Secondary CTA 또는 보조 버튼

### 빈 상태

- 아직 분석 결과가 없으면 "아직 AI 분석 결과가 없습니다" 메시지를 보여줍니다.
- 기본 동선은 활동 경험 상세로 돌아가 AI 분석 요청을 실행하는 흐름입니다.
- 단, 활동 경험 상세에서 진입한 경우에는 같은 경험을 기준으로 AI 분석 요청을 실행할 수 있습니다.

### 오류 상태

- AI 분석에 실패하면 실패 안내와 다시 시도 버튼을 보여줍니다.

## 5. AI 경험 추천 및 활용 화면

### 목적

사용자가 입력한 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문을 기준으로 저장된 경험 전체와 분석 결과를 참고해 가장 적합한 경험 1개를 추천하고, 추천 이유와 참고 문장을 제공합니다.

이 화면은 특정 경험 하나의 분석 결과에서 이어지는 화면이 아니라, 저장된 경험 전체와 분석 결과를 참고해 추천 결과를 제공하는 별도 흐름입니다.

### UI 구성 요소

- 활용 목적 선택
- 질문 / 문항 입력창
- 추천 요청 버튼
- 추천 경험 카드 1개
- 복사 버튼
- 저장 버튼

추천 경험 카드는 AI가 현재 활용 목적에 가장 적합하다고 판단한 경험 1개를 요약해 보여주는 영역입니다.

추천 경험 카드에는 아래 항목을 포함합니다.

- 추천 경험 제목
- 기간
- 관련 역량 태그
- 추천 이유
- 강조할 성과
- 활용 방향
- 자기소개서 / 포트폴리오 / 면접 답변 참고 문장

### 입력값

- 자기소개서 문항
- 포트폴리오 작성 목적
- 면접 질문
- 저장된 활동 경험 전체와 분석 결과

### 출력값

- 추천 경험
- 추천 이유
- 관련 역량 태그
- 강조할 성과
- 활용 방향
- 자기소개서 / 포트폴리오 / 면접 답변 참고 문장

1차 MVP에서는 가장 적합한 경험 1개만 추천합니다.

### 버튼 동작

- 추천 요청: 입력한 목적에 맞는 경험 추천과 참고 문장 생성을 요청
- 복사: 참고 문장을 클립보드에 복사
- 저장: 추천 결과를 브라우저 localStorage에 저장

추천 결과 저장은 MVP에 포함하지만, 첫 구현에서는 복사 기능보다 낮은 우선순위로 다룹니다. 여러 추천 후보 탐색 기능은 1차 MVP에서 제외합니다.

### 빈 상태

- 저장된 활동 경험이 없으면 먼저 활동 경험을 기록하라는 안내를 보여줍니다.
- 질문 / 문항이 비어 있으면 입력 안내를 보여줍니다.

### 오류 상태

- 추천 결과 생성에 실패하면 실패 안내와 다시 시도 버튼을 보여줍니다.
- 복사에 실패하면 직접 선택해서 복사할 수 있도록 안내합니다.

## Screen Flow

### 경험 기록 / 분석 흐름

```text
메인 화면
→ 활동 경험 목록
→ 활동 경험 상세
→ AI 경험 분석 결과
```

### 경험 작성 흐름

```text
메인 화면
→ 새 활동 경험 작성
→ 저장
→ 활동 경험 상세
→ AI 분석 요청
→ AI 경험 분석 결과
```

### 활용 추천 흐름

```text
메인 화면
→ AI 경험 추천 및 활용
→ 자기소개서 문항 / 포트폴리오 목적 / 면접 질문 입력
→ 저장된 경험 전체와 분석 결과를 기준으로 가장 적합한 경험 1개 추천
→ 참고 문장 복사 / 저장
```

## MVP Excluded Screens

1차 MVP에서는 아래 화면을 만들지 않습니다.

- 로그인 / 회원가입 화면
- 수료증 / 활동사진 첨부 화면
- 파일 첨부 관리 화면
- 결제 화면
- 커뮤니티 화면
- 관리자 화면
- 공개 프로필 화면
- 외부 서비스 자동 연동 설정 화면
- PDF 포트폴리오 자동 생성 화면
- 여러 추천 후보 탐색 화면

## Future Screen Memo

- 수료증 / 활동사진 첨부 UI는 2차 MVP에서 Supabase Storage 도입과 함께 확장 예정입니다.
- 로그인 / 회원가입, 마이페이지, 사용자별 저장 화면은 2차 MVP에서 Supabase Auth / Postgres 도입과 함께 검토합니다.
- 2차 MVP에서는 사용자의 활용 목적에 따라 1순위, 2순위, 3순위 경험 추천과 각 추천 이유 비교 UI를 검토합니다.
- Spring Boot, MySQL 또는 AWS RDS, AWS S3 기반 관리 화면은 UNIKER 이후 3차 확장에서 검토합니다.
