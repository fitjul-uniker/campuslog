# CampusLog Screen Spec

## Purpose

이 문서는 CampusLog 1차 MVP의 핵심 화면별 목적, UI 구성 요소, 입력값, 출력값, 버튼 동작, 빈 상태, 오류 상태를 정리합니다.

`/`의 3D 노트 표지를 제외한 모든 제품 화면은 대시보드와 같은 흰색 SaaS 작업 공간과 공통 앱 셸을 사용합니다. 작성·수정·활동 전체 화면·분석·추천·추천 기록 화면은 책 프레임이나 종이 줄무늬를 사용하지 않습니다.

## Premium 3D 노트 표지 화면

- `/`에서 검은 가죽 하드커버 노트를 실제 두께가 있는 3D 구조로 보여줍니다.
- 앞표지 텍스처에는 `black-leather-book.webp`의 가죽 결만 적용하고, 표지 외곽 형태는 3D geometry가 담당합니다.
- 3D Canvas는 장식 요소이며 중요한 문구나 기능을 포함하지 않습니다.
- 노트를 선택하면 `/dashboard`로 이동합니다.
- 로딩 중 정적 책 이미지를 먼저 노출하지 않습니다.
- 화면 전체 포인터 좌표에 반응하고 책 hover 또는 keyboard focus 시 미세하게 확대됩니다.
- `prefers-reduced-motion`에서는 포인터 추적, 반복 부유, hover 확대를 비활성화합니다.

## 0. 메인/경험 목록 대시보드

### 목적

사용자가 CampusLog 대시보드에 들어왔을 때 저장된 활동 제목을 바로 훑고, 목록을 유지한 채 선택한 활동의 상세를 같은 화면에서 확인할 수 있습니다. 우측 하단 `+` CTA로 새 경험을 작성하며, AI 추천 및 활용 화면은 데스크톱 좌측 메뉴에서 진입합니다.

1차 MVP에서는 로그인 / 회원가입 화면을 만들지 않습니다. 브랜드 첫 화면은 `/`의 3D 노트 표지이고, 기능 시작 화면은 `/dashboard`의 메인/경험 목록 대시보드입니다.

### UI 구성 요소

- 서비스 이름
- CountUp으로 표시되는 전체 활동 개수
- `검색`으로 표시되는 Gooey 스타일 제목 검색
- 중앙에 놓인 제목 전용 활동 목록
- 선택 전: 단일 목록 열
- 선택 후: 왼쪽으로 이동한 제목 목록과 오른쪽 활동 상세 패널
- 활동 상세 패널의 기간, 역할, 내용, 성과, 관련 링크, AI 분석 상태와 요약
- 전체 화면, 수정, AI 분석 결과 화면으로 이동하는 보조 액션
- 미분석 또는 재분석 필요 활동의 Border Beam형 AI 분석 요청 액션
- 우측 하단 `+` 새 경험 기록하기 CTA
- 빈 상태 안내
- 제목 행 형태의 로딩 상태
- 다시 시도할 수 있는 오류 상태

### 입력값

- 사용자가 선택한 활동 경험
- 활동 제목 검색어

### 출력값

- 서비스 이름
- 최근 수정순으로 정렬된 활동 제목 목록
- 선택한 활동의 인라인 상세 정보
- 저장된 경험이 없을 때의 안내 메시지

### 버튼 동작

- 활동 제목 선택: 목록은 유지한 채 데스크톱에서 왼쪽으로 이동하고 우측에 상세 패널 표시
- 상세 닫기 또는 `Escape`: 패널을 닫고 선택한 제목으로 키보드 초점 복귀
- 전체 화면으로 보기: 인라인 상세와 같은 구조의 활동 전체 화면으로 이동
- 수정 / AI 분석 결과: 해당 전용 화면으로 이동
- AI 분석 요청 / 다시 분석하기: 현재 화면에서 분석을 요청하고 상태와 결과 액션을 갱신
- 우측 하단 `+`: 활동 경험 작성 화면으로 이동

### 빈 상태

- 저장된 경험이 없으면 "아직 기록한 활동이 없습니다" 메시지를 보여주고, 우측 하단 `+` CTA와 첫 활동 기록하기 링크를 유지합니다.
- `+` 또는 첫 활동 기록하기를 선택하면 활동 경험 작성 화면으로 이동합니다.

### 오류 상태

- 저장된 데이터를 불러오지 못하면 다시 시도 안내를 보여줍니다.
- 검색 결과가 없으면 저장 데이터 빈 상태와 구분해 검색어 지우기를 제공합니다.

### 반응형 / 접근성

- 모바일에서는 제목 목록을 위에 유지하고 상세 패널을 아래에 쌓습니다.
- 제목 목록은 스크롤 가능한 고정 높이 영역과 상·하단 페이드로 다음 항목의 존재를 알립니다.
- 제목 행은 `button`으로 구현하며 `hover`, `active`, `focus-visible`, 선택 상태를 구분합니다.
- 목록 안에서는 `ArrowUp`, `ArrowDown`, `Home`, `End`로 초점을 이동할 수 있고 일반 `Tab` 순서는 유지합니다.
- `prefers-reduced-motion`에서는 위치 이동과 진입 변위를 제거하거나 즉시 전환합니다.

## 1. 활동 경험 작성 / 수정 화면

### 목적

사용자가 프로젝트, 공모전, 인턴, 대외활동 등 자신의 활동 경험을 직접 입력하거나 수정할 수 있게 합니다.

### UI 구성 요소

- 제목 입력
- 기간 입력
- 역할 입력
- 내용 입력
- 성과 입력
- 관련 링크 URL·설명 행
- 링크 추가 및 개별 삭제 버튼
- 저장 버튼
- 취소 버튼

1차 MVP에서는 수료증, 활동사진 등 파일 첨부 UI를 포함하지 않습니다.
작성 / 수정 화면에서는 AI 분석 요청 버튼을 제공하지 않습니다.

### 입력값

- 제목
- 기간
- 역할
- 내용
- 성과
- 관련 링크 URL
- 링크 설명(선택)

### 출력값

- 저장된 활동 경험

### 버튼 동작

- 저장: 입력한 경험을 저장하고 활동 경험 상세 화면으로 이동
- 취소: 입력 내용을 저장하지 않고 이전 화면 또는 메인/경험 목록 대시보드로 이동
- 링크 추가: 빈 URL·설명 행을 하나 추가하고 URL 입력으로 초점을 이동
- 링크 삭제: 선택한 링크 행만 저장 전 목록에서 제거

### 빈 상태

- 새 경험 작성 시 입력 필드는 비어 있는 상태로 보여줍니다.

### 오류 상태

- 필수 입력값이 비어 있으면 저장하지 않고 안내 메시지를 보여줍니다.
- 관련 링크 형식이 올바르지 않으면 링크 확인 안내를 보여줍니다.
- URL 없이 설명만 입력했거나 같은 URL을 중복으로 추가하면 해당 행에 오류를 표시하고 저장하지 않습니다.

## 2. 활동 경험 상세 화면

### 목적

대시보드 우측 활동 상세와 같은 정보 구조를 전체 화면 폭으로 확장해 원본 내용을 확인하고, 수정·삭제·AI 분석을 실행할 수 있게 합니다.

### UI 구성 요소

- 제목
- 기간
- 역할
- 내용
- 성과
- 설명, 도메인, 파비콘으로 구성된 관련 링크 목록
- 분석 완료된 경우 AI 생성 주요 키워드 / 역량 태그
- AI 분석 상태
- 수정 버튼
- AI 분석 요청 버튼
- AI 분석 결과 버튼
- 활동 목록 버튼
- 삭제 버튼

### 입력값

- 저장된 활동 경험

### 출력값

- 활동 경험 원본 내용
- 분석 완료된 경험의 AI 생성 주요 키워드 / 역량 태그
- AI 분석 상태

### 버튼 동작

- 활동 목록: 전체 화면의 가장 왼쪽 첫 번째 액션으로 두고 메인/경험 목록 대시보드로 이동
- AI 분석 요청 / AI 분석 결과: 두 번째 액션으로 두고 분석을 실행하거나 저장된 AI 경험 분석 결과 화면으로 이동
- 수정: 세 번째 액션으로 두고 활동 경험 작성 / 수정 화면으로 이동. 다른 액션보다 불필요하게 넓히지 않습니다.
- 삭제: 마지막 액션으로 두고 확인 후 연결 데이터를 함께 삭제

### 빈 상태

- 아직 AI 분석 결과가 없으면 "아직 AI 분석 결과가 없습니다" 메시지와 AI 분석 요청 버튼을 보여줍니다.

### 오류 상태

- 저장된 경험을 불러오지 못하면 메인/경험 목록 대시보드로 돌아가는 안내를 보여줍니다.

## 3. AI 경험 분석 결과 화면

### 목적

특정 활동 경험에 연결된 AI 분석 결과를 보여줍니다.

### UI 구성 요소

- 경험 요약 영역
- 핵심 역량 태그
- 주요 성과 목록
- 활용 가능한 키워드
- 분석 생성일
- 다시 분석 버튼
- 활동 경험 상세로 돌아가기 버튼
- 메인/경험 목록 대시보드로 돌아가기 버튼
- AI 경험 추천 및 활용으로 이동 보조 버튼

### 입력값

- 저장된 활동 경험

### 출력값

- 경험 요약
- 핵심 역량 태그
- 주요 성과
- 활용 가능한 키워드
- 분석 생성일

분석 결과는 특정 활동 경험에 연결된 분석 기록입니다. 분석 결과는 localStorage에 저장되어 사용자가 나중에 다시 확인할 수 있도록 합니다.
역량 태그는 고정 분류가 아니라, 대표 예시를 참고해 AI가 생성하는 분석 결과입니다.
대표 예시는 문제 해결력(문제를 발견하고 원인을 분석해 해결한 경험), 협업/커뮤니케이션(팀원과 역할을 나누고 의견을 조율한 경험), 실행력/주도성(목표를 정하고 실제 결과물까지 만들어낸 경험), 분석력/데이터 활용력(자료, 피드백, 수치, 사용자 반응 등을 바탕으로 판단한 경험)입니다.

### 버튼 동작

- 다시 분석: 이미 분석 결과가 있는 경우, 같은 경험을 기준으로 AI 분석을 다시 요청
- 활동 경험 상세로 돌아가기: 해당 활동 경험 상세 화면으로 이동하는 Primary CTA
- 메인/경험 목록 대시보드로 돌아가기: 메인/경험 목록 대시보드로 이동하는 Primary CTA
- AI 경험 추천 및 활용으로 이동: AI 경험 추천 및 활용 화면으로 이동할 수 있는 Secondary CTA 또는 보조 버튼

### 빈 상태

- 아직 분석 결과가 없으면 "아직 AI 분석 결과가 없습니다" 메시지를 보여줍니다.
- 활동 경험 상세로 돌아가 AI 분석 요청을 실행하도록 안내합니다.

### 오류 상태

- AI 분석에 실패하면 실패 안내와 다시 시도 버튼을 보여줍니다.

## 4. AI 경험 추천 및 활용 화면

### 목적

사용자가 입력한 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문을 기준으로 저장된 경험 전체와 분석 결과를 참고해 가장 적합한 경험 1개를 추천하고, 추천 이유와 참고 문장을 제공합니다.

이 화면은 특정 경험 하나의 분석 결과에서 이어지는 화면이 아니라, 저장된 경험 전체와 분석 결과를 참고해 추천 결과를 제공하는 별도 흐름입니다.

### UI 구성 요소

- 활용 목적 선택
- 질문 / 문항 입력창
- 추천 요청 버튼
- 추천 경험 카드 1개
- 복사 버튼
- 저장 버튼

추천 경험 카드는 AI가 현재 활용 목적에 가장 적합하다고 판단한 경험 1개를 요약해 보여주는 영역입니다.

추천 경험 카드에는 아래 항목을 포함합니다.

- 추천 경험 제목
- 기간
- 관련 역량 태그
- 추천 이유
- 강조할 성과
- 활용 방향
- 자기소개서 / 포트폴리오 / 면접 답변 참고 문장

### 입력값

- 자기소개서 문항
- 포트폴리오 작성 목적
- 면접 질문
- 저장된 활동 경험 전체와 분석 결과

### 출력값

- 추천 경험
- 추천 이유
- 관련 역량 태그
- 강조할 성과
- 활용 방향
- 자기소개서 / 포트폴리오 / 면접 답변 참고 문장

1차 MVP에서는 가장 적합한 경험 1개만 추천합니다.

### 버튼 동작

- 추천 요청: 입력한 목적에 맞는 경험 추천과 참고 문장 생성을 요청
- 복사: 참고 문장을 클립보드에 복사
- 저장: 추천 결과를 브라우저 localStorage에 저장

새 추천 결과가 생성되면 결과 영역 상단으로 자동 스크롤합니다. `prefers-reduced-motion`에서는 부드러운 이동 대신 즉시 이동하고, 키보드 초점은 강제로 옮기지 않습니다.

추천 결과의 활동 제목은 크고 짙은 본문 제목으로, `활용 목적`, `추천 이유`, `강조할 성과` 같은 항목명은 작고 옅은 라벨로, 실제 답변은 중간 크기의 짙은 본문으로 구분합니다.

추천 결과 저장은 MVP에 포함하지만, 첫 구현에서는 복사 기능보다 낮은 우선순위로 다룹니다. 여러 추천 후보 탐색 기능은 1차 MVP에서 제외합니다.

### 빈 상태

- 저장된 활동 경험이 없으면 먼저 활동 경험을 기록하라는 안내를 보여줍니다.
- 질문 / 문항이 비어 있으면 입력 안내를 보여줍니다.

### 오류 상태

- 추천 결과 생성에 실패하면 실패 안내와 다시 시도 버튼을 보여줍니다.
- 복사에 실패하면 직접 선택해서 복사할 수 있도록 안내합니다.

## 4-1. 추천 기록 화면

### 목적

사용자가 이전에 입력한 활용 목적과 질문 / 문항, AI가 추천한 경험과 활용 문장을 나중에 다시 확인할 수 있도록 저장된 추천 결과를 목록으로 보여줍니다.

### UI 구성 요소

- 추천 기록 목록
- 추천 기록 개수
- `검색`으로 표시되는 Gooey 스타일 검색
- 선택 전 중앙의 상세형 목록
- 선택 후 왼쪽 목록과 오른쪽 추천 상세 영역
- 새 추천 받기 버튼
- 대시보드로 돌아가기 버튼

추천 기록 목록에는 아래 항목을 포함합니다.

- 활용 목적
- 질문 / 문항
- 추천 경험 제목
- 추천 생성일

추천 기록 상세 영역에는 AI 경험 추천 및 활용 화면의 추천 결과 카드와 동일하게 추천 경험, 추천 이유, 관련 태그, 강조할 성과, 활용 방향, 참고 문장을 표시합니다.

추천된 원본 활동으로 이동하는 링크 문구는 `활동`으로 표시합니다.

추천 기록을 선택해도 목록은 유지됩니다. 모바일에서는 목록을 위에, 상세를 아래에 쌓고 상세 닫기 후 선택 행으로 초점을 복귀합니다.

### 빈 상태

- 저장된 추천 기록이 없으면 먼저 AI 추천을 요청하라는 안내를 보여줍니다.
- AI 추천 화면으로 이동하는 버튼을 보여줍니다.
- 검색 결과가 없으면 검색어 지우기 액션을 보여줍니다.

## Screen Flow

### 경험 기록 / 분석 흐름

```text
메인/경험 목록 대시보드
→ 기존 활동 제목 선택
→ 목록을 유지한 인라인 활동 상세
→ AI 분석 요청 또는 전체 화면으로 보기
→ AI 경험 분석 결과
```

### 경험 작성 흐름

```text
메인/경험 목록 대시보드
→ 우측 하단 새 경험 기록하기
→ 활동 경험 작성
→ 저장
→ 활동 전체 화면
→ AI 분석 요청
→ AI 경험 분석 결과
```

### 빈 상태 흐름

```text
메인/경험 목록 대시보드
→ 저장된 경험 없음
→ 우측 하단 새 경험 기록하기
→ 활동 경험 작성
```

### 활용 추천 흐름

```text
메인/경험 목록 대시보드
→ AI 경험 추천 및 활용
→ 자기소개서 문항 / 포트폴리오 목적 / 면접 질문 입력
→ 저장된 경험 전체와 분석 결과를 기준으로 가장 적합한 경험 1개 추천
→ 참고 문장 복사 / 저장
→ 추천 기록 검색 / 선택
→ 목록을 유지한 추천 상세
```

## MVP Excluded Screens

1차 MVP에서는 아래 화면을 만들지 않습니다.

- 로그인 / 회원가입 화면
- 로그인 / 회원 전환 목적의 별도 서비스 소개 랜딩 화면
- 수료증 / 활동사진 첨부 화면
- 파일 첨부 관리 화면
- 결제 화면
- 커뮤니티 화면
- 관리자 화면
- 공개 프로필 화면
- 외부 서비스 자동 연동 설정 화면
- PDF 포트폴리오 자동 생성 화면
- 여러 추천 후보 탐색 화면
