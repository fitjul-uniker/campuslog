# 부족한 정보 질문 MorphSurface 디자인

## 목적

AI 분석 결과의 `부족한 정보` 질문을 긴 입력 카드 목록이 아니라, 질문별로
간결하게 닫혀 있다가 답변할 때 제자리에서 확장되는 MorphSurface로 바꾼다.
사용자는 여러 질문의 상태를 빠르게 훑고 필요한 질문 하나에만 집중할 수 있어야
한다.

## 레퍼런스

- Cult UI `MorphSurface`
  - https://www.cult-ui.com/docs/components/morph-surface
  - MIT 라이선스의 공식 구현 구조를 참고한다.
- 공식 구현의 핵심인 `motion/react` spring, `layoutId` 기반 indicator 이동,
  클릭 밖 닫기, 자동 포커스, Escape 닫기, 키보드 제출을 유지한다.
- CampusLog에 이미 `motion/react`와 `lucide-react`가 있으므로 동일한 동작에
  추가 패키지가 필요하지 않다. 구현 중 공식 동작을 재현하기 위해 꼭 필요한
  의존성이 확인될 때만 추가한다.

## 선택한 방식

질문마다 독립된 MorphSurface를 제공하되 한 번에 하나만 열리게 한다.

- 여러 질문을 동시에 펼치는 방식은 세로 길이가 급격히 늘어나는 문제가 있다.
- 일반 아코디언은 구현은 단순하지만 레퍼런스의 표면 크기와 indicator가
  자연스럽게 변하는 감각을 살리기 어렵다.
- 한 번에 하나만 여는 방식은 목록의 밀도를 유지하면서 현재 답변 대상에 초점을
  준다.

## 닫힌 상태

- 한 줄 표면에 도움말 아이콘, 질문 제목, 질문 유형, 답변 상태를 표시한다.
- 질문 문장이 길면 한 줄 말줄임 처리하고 접근 가능한 이름에는 전체 문장을
  유지한다.
- 미답변은 중립 indicator, 답변 완료는 체크 indicator로 구분한다.
- 표면 전체가 열기 버튼이며 최소 높이 44px와 `focus-visible`을 유지한다.

## 열린 상태

- 같은 표면이 너비와 높이를 늘리며 답변 폼으로 변한다.
- 상단에는 질문 유형과 전체 질문을 표시한다.
- 질문 이유는 보조 문구로 한 단계 낮게 표시한다.
- 저장된 답변이 있으면 textarea에 그대로 채워 수정할 수 있게 한다.
- 하단에는 작성 상태 또는 마지막 저장 시각과 `답변 저장` / `수정 저장` CTA를
  배치한다.
- 저장 실패 안내는 해당 질문 안에서만 표시한다.

## 모션과 상호작용

- `motion/react` spring과 공유 `layoutId`를 사용해 indicator가 닫힌 트리거와
  열린 표면 사이를 이동하게 한다.
- 새 질문을 열면 기존 질문은 닫히고 새 질문이 열린다.
- 열릴 때 textarea로 포커스를 이동한다.
- Escape 또는 바깥 클릭으로 닫는다.
- macOS에서는 `Command + Enter`, Windows/Linux에서는 `Ctrl + Enter`로
  저장한다.
- 저장 성공 시 체크와 `저장 완료`를 표시한 뒤 표면을 자연스럽게 접는다.
- `prefers-reduced-motion`에서는 크기 전환 시간을 최소화하고 콘텐츠를 즉시
  전환한다.
- 닫힐 때 빈 표면, 잘린 텍스트, 레이아웃 점프가 보이지 않도록 콘텐츠
  opacity와 표면 크기 전환 순서를 맞춘다.

## 컴포넌트 경계

### `MorphSurface`

- 공용 UI 컴포넌트로 닫힘/열림 상태, spring, indicator 이동, 바깥 클릭,
  키보드 닫기와 포커스 이동을 담당한다.
- CampusLog 질문 전용 문구나 저장소 로직을 알지 않는다.
- controlled 상태와 custom trigger/content 렌더링을 지원해 질문 목록에서
  한 번에 하나만 열도록 제어한다.

### `AnalysisGapAnswerList`

- 기존 `answerItems`, draft, 저장 중, 오류, 저장 완료 상태를 그대로 관리한다.
- 열린 질문 id만 추가로 관리한다.
- MorphSurface에 질문별 trigger와 content를 전달한다.
- 기존 `handleSaveAnswer`의 repository 저장 순서와 데이터 계약은 변경하지
  않는다.

## 데이터와 오류 처리

- 분석 `evidenceGaps`, `experience_followups`, 분석 gap answer 저장 구조는
  변경하지 않는다.
- API, schema, repository 메서드와 local/Supabase 저장 형식은 변경하지 않는다.
- 빈 답변은 저장하지 않고 기존 인라인 오류를 유지한다.
- 저장 실패 시 표면을 닫지 않으며 입력값을 보존한다.
- 저장 성공 때만 완료 indicator와 자동 닫힘을 실행한다.
- 답변 로딩 실패와 전체 로딩 상태는 기존 방식으로 유지한다.

## 반응형과 접근성

- 데스크톱에서는 부모 분석 패널 너비를 채운다.
- 모바일에서는 고정 pixel 너비를 쓰지 않고 가용 너비 100%를 사용한다.
- 긴 질문, 긴 답변, 200% 텍스트 확대에서도 가로 스크롤이 생기지 않게 한다.
- 트리거는 실제 `button`, 답변 영역은 실제 `textarea`와 `button`을 사용한다.
- `aria-expanded`, `aria-controls`, 상태 텍스트와 오류 `role="alert"`를
  제공한다.
- 바깥 클릭 닫기는 키보드 사용자의 포커스를 잃게 하지 않는다.

## 검증 기준

- 처음에는 모든 질문이 한 줄 표면으로 보인다.
- 질문을 누르면 해당 질문만 자연스럽게 확장된다.
- 다른 질문을 누르면 이전 질문이 닫히고 새 질문이 열린다.
- 기존 답변 저장과 수정 저장이 동일하게 동작한다.
- 저장 실패 시 입력과 열린 상태가 유지된다.
- 저장 성공 시 완료 상태를 표시하고 닫힌 표면으로 돌아간다.
- Escape, 바깥 클릭, Command/Ctrl+Enter, 키보드 탐색이 동작한다.
- 모바일과 데스크톱에서 잘림과 가로 스크롤이 없다.
- reduced motion에서 반복되거나 큰 크기 모션이 제거된다.
- 관련 구조 테스트, lint, typecheck, `git diff --check`, 실제 브라우저 확인을
  통과한다.

## 범위 밖

- AI가 만드는 부족 정보 질문의 내용과 개수 변경
- 답변 저장 schema/API 변경
- 추천이나 답변 초안 로직 변경
- 부족 정보가 아닌 다른 카드 전체의 일괄 MorphSurface 전환
