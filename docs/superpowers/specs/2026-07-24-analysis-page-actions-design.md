# AI 분석 페이지 액션 재배치 디자인

## 목적

독립 AI 분석 화면의 복귀·재실행·교차 이동 액션을 사용 시점에 맞게 분리해 결과 하단의 버튼 밀도를 낮춥니다.

## 최종 배치

- 페이지 헤더 오른쪽:
  - `활동 경험 상세로 돌아가기`
  - `나의 활동으로 돌아가기`
- 분석 결과 맨 아래 왼쪽:
  - `다시 분석하기`
- 제거:
  - `AI 기반 활동 추천`

분석 결과가 없는 상태도 페이지 헤더에 두 복귀 액션을 제공하고, 빈 결과 카드 하단에는 `AI 분석 요청`만 남깁니다.

## 위계

- 두 복귀 링크는 사용자가 결과를 끝까지 읽지 않아도 접근할 수 있는 페이지 탐색 액션이므로 헤더에 둡니다.
- `다시 분석하기`와 `AI 분석 요청`은 현재 경험의 분석 작업을 실행하는 문맥 액션이므로 콘텐츠 하단에 단독 배치합니다.
- `AI 기반 활동 추천`은 전역 메뉴에서 접근할 수 있고 경험 분석 완료와 직접 연결되지 않으므로 이 화면에서 제거합니다.

## 동작과 접근성

- 기존 href, 분석 요청 handler, loading, disabled, `aria-busy` 계약은 유지합니다.
- 상단 복귀 링크는 기존 중립 버튼 표현과 44px 조작 영역을 유지합니다.
- 하단 분석 실행 버튼은 기존 animated gradient 표현을 유지하고 왼쪽 정렬합니다.
- 모바일에서는 상단 복귀 링크가 자연스럽게 줄바꿈되며 가로 overflow를 만들지 않아야 합니다.

## 영향 범위

- `ExperienceAnalysisClient`의 액션 렌더링과 필요한 레이아웃 스타일만 변경합니다.
- API, schema, repository, 저장 데이터와 분석 결과 구조는 변경하지 않습니다.
