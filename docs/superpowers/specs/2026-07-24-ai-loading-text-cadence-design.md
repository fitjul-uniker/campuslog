# AI Loading Text Cadence Design

## 목적

Strands 기반 공용 AI 처리 화면의 상태 문구가 너무 빠르게 바뀌어 산만하게 보이는 문제를 줄이고, 대기 상태에 맞는 말줄임 표현을 일관되게 적용한다.

## 승인된 변경

- `AIProcessingPanel`이 `AITextLoading`에 전달하는 전환 간격을 1.8초에서 2.4초로 늘린다.
- 공용 AI 처리 화면에 표시하는 title, status, step, description, 장기 대기 문구는 화면 표시 직전에 끝맺음을 `...`로 통일한다.
- 기존 문구가 `.`, `..`, `...` 또는 `…`로 끝나면 해당 끝맺음만 제거한 뒤 ASCII 점 세 개를 붙여 중복을 막는다.
- 원본 상태 메시지, SSE / NDJSON 이벤트, API response, 저장 데이터는 변경하지 않는다.
- 빈 문자열 제거와 중복 문구 제거는 기존 동작을 유지한다.
- 공용 취소 액션은 44px 클릭 영역과 아이콘·글자를 유지하면서 배경·테두리·그림자를 제거한다.
- 추천 요청의 `REQUEST_CANCELLED`는 오류 alert를 만들지 않고 입력만 유지한다. 실제 실패는 기존 오류 alert를 유지한다.

## 적용 범위

공용 `AIProcessingPanel`을 사용하는 경험 분석·재분석, AI 추천·JD 분석, 활동 완료 경험 합성, 답변 초안 첫 결과 대기에 동일하게 적용한다.

## 접근성 및 모션

- 기존 `aria-live`, dialog, 취소 버튼과 reduced motion 동작을 유지한다.
- 문구 전환 애니메이션의 형태와 지속 시간은 변경하지 않고, 문구가 머무는 간격만 늘린다.

## 검증

- 구조 테스트에서 `interval={2_400}` 적용을 확인한다.
- 문구 정규화 함수가 일반 마침표와 기존 말줄임을 정확히 `...` 하나로 바꾸는지 확인한다.
- lint, typecheck와 관련 테스트를 실행한다.
- 실제 AI 처리 화면에서 문구 전환 속도와 `...` 끝맺음을 확인한다.
- 추천 취소 뒤 입력 유지, alert 0개, overlay와 body scroll 복구를 확인한다.
