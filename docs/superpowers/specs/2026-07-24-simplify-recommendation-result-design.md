# AI 추천 결과 정보 간소화 디자인

## 목적

현재 추천 결과와 저장된 추천 기록 상세에서 활용도가 낮고 화면을 길게 만드는 보조 정보를 제거해 추천 경험과 근거에 집중합니다.

## 적용 화면

- `/recommend`의 새 AI 추천 결과
- `/recommend/history`의 저장된 추천 기록 상세

두 화면이 함께 사용하는 `RecommendationResult`의 표시 구조만 변경합니다.

## 제거할 정보

- 질문 또는 요구사항 분석의 `필수 역량`
- 질문 또는 요구사항 분석의 `키워드`
- 질문 또는 요구사항 분석의 `제약`
- `1순위 요약` 전체
- `참고 문장`과 복사 버튼·복사 상태 안내

`우대 역량`과 질문 또는 요구사항의 의도 설명은 유지합니다. 추천 Top 3, 추천 이유와 직접 근거, 부족 정보, 과장 주의점, 활용 방향, JD 분석과 답변 생성 흐름도 유지합니다.

## 데이터와 로직

- 추천 API 응답 구조는 변경하지 않습니다.
- Supabase 및 localStorage 저장 구조를 변경하지 않습니다.
- 기존 추천 기록의 역직렬화와 하위 호환을 유지합니다.
- 제거 대상 필드는 데이터에 남아 있어도 화면에서만 렌더링하지 않습니다.
- 추천 생성, 저장, 검색, 선택, 답변 초안 생성 로직은 변경하지 않습니다.

## 접근성과 상태

- `참고 문장`을 제거하면서 해당 복사 버튼과 성공·실패 상태도 함께 제거해 보이지 않는 조작 요소가 남지 않게 합니다.
- 제거 후 제목 단계와 구획 사이 여백이 자연스럽게 이어져야 합니다.
- 모바일과 데스크톱에서 빈 그리드나 불필요한 구분선이 남지 않아야 합니다.

## 검증

- 현재 추천과 추천 기록 상세에서 제거 대상 문구가 렌더링되지 않는지 구조 테스트로 확인합니다.
- 추천 Top 3, 근거, JD 분석과 답변 생성 영역이 유지되는지 확인합니다.
- lint, typecheck, 관련 테스트와 브라우저 화면을 확인합니다.

## 범위 제외

- 추천 prompt와 structured output 변경
- 추천 데이터 migration 또는 삭제
- 추천 품질 및 순위 로직 변경
- 다른 AI 분석 화면의 키워드 표시 변경
