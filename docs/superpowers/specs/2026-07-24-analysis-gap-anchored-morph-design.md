# 부족 정보 질문 Anchored Morph 디자인

## 배경

현재 부족 정보 질문은 열린 `MorphSurface` 전체를 위로 12px 이동합니다. 질문 목록의 간격도 12px이므로 두 번째 이후 질문을 열면 열린 표면의 상단과 이전 질문의 하단이 같은 좌표에 놓이고, 그림자와 경계가 겹쳐 보입니다. 또한 루트에 적용된 Motion `layout` spring이 열린 질문과 인접 질문을 동시에 크기 변형해 질문을 바꿀 때 표면이 서로 밀고 지나가는 느낌을 만듭니다.

본문은 `y: -10 → 0`으로 나타나므로 Cult UI MorphSurface 레퍼런스의 아래에서 위로 떠오르는 감각과 반대로 위에서 아래로 내려옵니다.

## 목표

- 질문 표면이 서로 침범하지 않고 질문 사이의 간격을 유지합니다.
- 펼친 본문이 아래에서 위로 올라오며 표면 안에 자연스럽게 정착합니다.
- 질문을 바꿀 때 기존 본문 닫힘이 먼저 시작되고 새 본문이 뒤이어 열립니다.
- 인접 질문은 scale 변형 없이 열리는 높이에 맞춰 부드럽게 이동합니다.
- 기존의 한 번에 하나만 열기, 바깥 클릭, Escape, 초점 이동, 키보드 저장과 저장 상태 계약을 유지합니다.

## 승인된 방향

### Anchored Morph

질문 trigger와 표면의 상단 좌표를 고정합니다. 열린 표면 전체를 음의 y축으로 이동하지 않고, 흰색 표면의 모서리와 그림자만 열린 상태로 전환합니다.

본문은 높이 `0 → auto` reveal 안에서 잘리고, 내부 내용은 `y: 12 → 0`과 opacity 전환으로 아래에서 위로 들어옵니다. 닫을 때는 reveal 높이를 먼저 줄이며 새 질문의 reveal에는 짧은 지연을 적용해 두 편집기가 동시에 크게 보이지 않게 합니다.

### 간격과 레이어

- 질문 목록 기본 간격: 16px
- 열린 표면: `position: relative`, `z-index: 1`
- 루트 `layout` scale animation 제거
- reveal이 높이를 애니메이션하므로 아래 질문은 브라우저 reflow를 따라 위치만 이동
- 강한 elevation 대신 기존 중립 그림자를 유지

### 모션

- 열림 reveal: 약 260ms, 60ms 지연
- 닫힘 reveal: 약 180ms, 지연 없음
- 본문: `y: 12 → 0`, opacity `0 → 1`
- Chevron 회전과 indicator 표현은 유지
- reduced motion: 높이·위치·spring을 0.01초 전환으로 축소

### 반응형

모바일에서도 표면 전체 translate를 사용하지 않습니다. 고정 폭 없이 가용 폭을 채우고 저장 버튼의 full-width 규칙을 유지합니다.

## 접근성

- trigger의 `aria-expanded`, `aria-controls`를 유지합니다.
- 열림 후 textarea 포커스는 `preventScroll: true`를 유지합니다.
- Escape와 바깥 클릭 닫기를 유지합니다.
- reduced motion에서는 부상·본문 이동을 제거합니다.

## 영향 범위

- UI와 전환만 변경합니다.
- 분석, 보완 답변 저장, 추천 반영, API, schema, repository 계약은 변경하지 않습니다.

## 검증 기준

1. 두 번째 질문이 열려도 이전 질문과 최소 16px의 시각 간격이 유지됩니다.
2. 루트 표면에 음의 y축 translate와 layout scale이 적용되지 않습니다.
3. 본문은 아래에서 위로 나타납니다.
4. 질문 전환 중 두 편집기가 겹치지 않습니다.
5. 데스크톱과 모바일에서 가로 잘림이 없습니다.
6. 키보드, focus-visible, Escape, 바깥 클릭과 reduced motion 계약이 유지됩니다.
