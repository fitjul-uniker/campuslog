# CampusLog User Flow

## Purpose

이 문서는 CampusLog 1차 MVP에서 대학생 사용자가 활동 경험을 기록하고, AI 분석 결과를 바탕으로 자기소개서·포트폴리오·면접 준비에 활용하는 전체 흐름을 정리합니다.

## MVP Included Flow

1차 MVP의 핵심 기능 흐름은 로그인 없이 `/dashboard`의 메인/경험 목록 대시보드에서 시작하며, 브라우저 localStorage에 활동 경험과 분석 / 추천 결과를 저장하는 기준으로 설계합니다. `/`에는 회원 전환형 랜딩이 아닌 premium 3D 노트 표지가 선행하고, 노트를 선택하면 `/dashboard`로 진입합니다.

1. **메인/경험 목록 대시보드에서 활동 경험 기록 시작**
   - 사용자는 흰색 대시보드에서 서비스 이름과 저장된 활동의 제목 목록을 확인합니다.
   - 활동 개수는 CountUp으로 표시되며, `검색`을 열어 활동 제목을 빠르게 필터링할 수 있습니다.
   - 활동 제목을 선택하면 목록은 사라지지 않고 데스크톱에서 왼쪽으로 이동하며, 오른쪽에 해당 활동의 상세 패널이 나타납니다.
   - 상세 패널에서 기간, 역할, 내용, 성과, 관련 링크와 AI 분석 요약을 확인하고 필요하면 `전체 화면으로 보기`로 이동합니다.
   - 대시보드 우측 하단의 `+` CTA를 통해 활동 경험 작성 화면으로 이동합니다.

2. **활동 정보 입력**
   - 제목, 기간, 역할, 내용, 성과를 입력하고, 관련 자료는 링크별 URL과 설명으로 나누어 저장합니다.
   - 활동 한 줄 주제 또는 핵심 주제는 선택 입력 항목으로 두며, AI가 경험을 분류하거나 추천할 때 참고할 수 있습니다.
   - 1차 MVP에서는 첨부파일 대신 관련 링크를 하나씩 추가하며, URL과 해당 자료의 설명을 함께 남깁니다.
   - 저장 후 활동 경험 상세 화면으로 이동합니다.

3. **활동 경험 수정 / 삭제**
   - 사용자는 활동 경험 상세 화면에서 이미 작성한 활동 경험을 수정할 수 있습니다.
   - 사용자는 작성한 활동 경험을 삭제할 수 있으며, 삭제 전에는 실수 방지를 위한 확인 절차를 거칩니다.
   - 분석이 완료된 활동은 목록 또는 상세 화면에서 "분석 완료" 상태로 볼 수 있습니다.
   - 분석 완료 후 활동 내용을 수정하면 기존 분석 결과와 내용이 달라질 수 있으므로 "재분석 필요" 상태가 될 수 있습니다.

4. **AI 분석 요청**
   - 사용자는 대시보드의 인라인 활동 상세 또는 활동 전체 화면에서 저장된 경험을 바탕으로 AI 분석 요청을 실행합니다.
   - 미분석 활동은 분석 결과 액션 위치에 Border Beam형 `AI 분석 요청` 버튼을 보여줍니다.
   - 작성 / 수정 화면에서는 AI 분석 요청을 바로 실행하지 않습니다.
   - "재분석 필요" 상태의 활동도 다시 AI 분석 요청을 실행할 수 있습니다.
   - AI는 경험 요약, 핵심 역량 태그, 주요 성과, 세부 키워드 또는 활용 가능한 키워드, 분석 생성일을 분석 결과로 정리합니다.

5. **분석 결과 확인**
   - 분석 결과는 특정 활동 경험에 연결된 기록이며, localStorage에 저장되어 나중에 다시 확인할 수 있습니다.
   - 사용자는 AI가 정리한 핵심 역량 태그, 성과, 세부 키워드를 확인하고 자신의 경험을 다시 이해합니다.
   - 핵심 역량 태그는 자기소개서 / 면접에 활용하기 좋은 넓은 역량 범주로 유지합니다.
   - 핵심 역량 태그는 고정 분류가 아니라, 대표 예시를 참고해 AI가 생성하는 분석 결과입니다.
   - 대표 예시는 다음과 같습니다.
     - 문제 해결력: 문제를 발견하고 원인을 분석해 해결한 경험
     - 협업/커뮤니케이션: 팀원과 역할을 나누고 의견을 조율한 경험
     - 실행력/주도성: 목표를 정하고 실제 결과물까지 만들어낸 경험
     - 분석력/데이터 활용력: 자료, 피드백, 수치, 사용자 반응 등을 바탕으로 판단한 경험
   - 세부 키워드 또는 활용 가능한 키워드는 포트폴리오 구성과 추천 정확도 개선에 활용하기 좋은 구체 기술·활동 키워드입니다.
   - 세부 키워드 예시는 AI 개발, Python, Git 협업, 데이터 구조화, MVP 개발 등입니다.

6. **활용 목적 입력**
   - 사용자는 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문, 기타 문항 / 활용 목적 등을 입력합니다.
   - 기타 문항 / 활용 목적에는 장학금 신청 문항, 대외활동 지원 문항, 동아리 지원 문항, 블로그 회고 글 작성 목적 등이 포함될 수 있습니다.

7. **AI 경험 추천 및 문장 생성**
   - AI 경험 추천 및 활용 화면은 특정 경험 하나의 분석 결과에서 이어지는 화면이 아닙니다.
   - AI는 1차 MVP 기준으로 저장된 경험 전체와 분석 결과를 참고해 활용 목적에 가장 적합한 경험 1개만 추천합니다.
   - 추천 결과에는 추천 경험, 추천 이유, 관련 핵심 역량 태그, 강조할 성과, 활용 방향, 자기소개서 / 포트폴리오 / 면접 답변 / 기타 활용 목적 참고 문장이 포함됩니다.
   - 사용자는 추천 결과를 바탕으로 자기소개서 초안, 포트폴리오 문장, 면접 답변 또는 기타 목적의 참고 문장을 확인합니다.
   - 새 추천 결과가 생성되면 화면은 결과 상단으로 자동 이동하며, 키보드 초점은 사용자가 조작하던 위치에 유지합니다.

8. **복사 또는 저장**
   - 사용자는 AI가 제안한 참고 문장을 복사해 실제 지원 준비에 활용합니다.
   - 추천 결과 저장은 MVP에 포함하지만, 첫 구현에서는 복사 기능보다 낮은 우선순위로 다룹니다.
   - 추천 기록 화면에서는 활용 목적, 추천 경험, 질문을 검색하고 목록을 유지한 채 선택한 추천 상세를 확인합니다.

## Core Flow

경험 기록 / 분석 흐름과 활용 추천 흐름은 메인/경험 목록 대시보드를 공통 시작점으로 사용하는 별도 진입 흐름으로 봅니다.

### 경험 기록 / 분석 흐름

```text
메인/경험 목록 대시보드
→ 우측 하단 + 또는 기존 활동 제목 선택
→ 목록을 유지한 인라인 활동 상세
→ AI 분석 요청 또는 전체 화면으로 보기
→ AI 경험 분석 결과 저장
→ 분석 완료 상태 확인
```

분석 완료 후 활동 내용을 수정한 경우 기존 분석 결과와 내용이 달라질 수 있어 "재분석 필요" 상태로 표시할 수 있습니다.
활동 경험 삭제는 확인 절차를 거친 뒤 실행합니다.

### 빈 상태 흐름

```text
메인/경험 목록 대시보드
→ 저장된 경험 없음
→ 우측 하단 + 또는 첫 활동 기록하기
→ 활동 경험 작성 화면
```

### 활용 추천 흐름

```text
데스크톱 좌측 메뉴
→ AI 추천 및 활용
→ 자기소개서 문항 / 포트폴리오 작성 목적 / 면접 질문 / 기타 문항 또는 활용 목적 입력
→ 저장된 경험 전체와 분석 결과 기준으로 가장 적합한 경험 1개 추천
→ 참고 문장 복사 / 추천 결과 저장
→ 추천 기록 검색 / 선택
→ 목록을 유지한 추천 상세 확인
```

## MVP Excluded Flow

1차 MVP에서는 아래 흐름을 포함하지 않습니다.

- 로그인 / 회원가입
- 로그인 / 회원 전환 목적의 별도 서비스 소개 랜딩 화면
- 학교 인증
- 결제
- 커뮤니티
- GitHub 자동 연동
- 블로그 / Notion 자동 연동
- PDF 포트폴리오 자동 생성
- 고급 추천 알고리즘
- 여러 추천 후보 탐색 (1차 MVP 제외, 2차 MVP 확장 후보)
- 모바일 앱 전용 흐름
- 로그인 기반 사용자별 활동 기록 저장
- 수료증 / 활동사진 / PDF / 산출물 등 첨부파일 저장
- 첨부파일 미리보기 / 용량 관리 / 파일 삭제 / 파일 보안 처리
- 2차 MVP 이전의 Supabase Auth / Postgres / Storage 기반 저장 흐름
- UNIKER 이후 개인 Backend Portfolio Edition용 회원 / 데이터베이스 / 파일 저장 흐름

첨부파일 저장은 localStorage 기준 1차 MVP에서는 파일 저장, 용량, 삭제, 보안, 미리보기 처리가 범위를 키우므로 제외합니다.
1차 MVP에서는 URL과 설명을 가진 관련 링크 목록으로 대체합니다.

## Future Flow

- 2차 MVP에서는 Next.js Full Stack + Supabase 기반으로 외부 사용자 전환용 서비스 소개 랜딩, Supabase Auth / Postgres 기반 로그인 저장과 Supabase Storage 기반 수료증 / 활동사진 / PDF / 산출물 첨부파일 저장, 미리보기, 삭제 흐름을 검토합니다.
- 1차 MVP에서도 Three.js, React Three Fiber, GSAP, Framer Motion, WebGL, shader, particle, scroll-based animation, micro interaction 같은 표현은 사용자 흐름을 강화하는 경우 사용할 수 있습니다.
- 2차 MVP에서는 사용자의 활용 목적에 따라 적합한 경험을 1순위, 2순위, 3순위까지 추천하고, 각 경험의 추천 이유를 비교해 보여주는 UI를 검토합니다.
- UNIKER 이후 개인 Backend Portfolio Edition에서는 Spring Boot REST API, MySQL 또는 AWS RDS, AWS S3 기반 백엔드 포트폴리오 확장 흐름을 별도 검토합니다.
