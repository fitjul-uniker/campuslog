# CampusLog User Flow

## Purpose

이 문서는 CampusLog 1차 MVP에서 대학생 사용자가 활동 경험을 기록하고, AI 분석 결과를 바탕으로 자기소개서·포트폴리오·면접 준비에 활용하는 전체 흐름을 정리합니다.

## MVP Included Flow

1차 MVP 흐름은 로그인 없이 브라우저 localStorage에 활동 경험과 분석 / 추천 결과를 저장하는 기준으로 설계합니다.

1. **활동 경험 기록 시작**
   - 사용자는 프로젝트, 공모전, 인턴, 대외활동 등 기록할 경험을 선택하거나 새로 작성합니다.

2. **활동 정보 입력**
   - 제목, 기간, 역할, 내용, 성과, 관련 링크를 입력하고 저장합니다.

3. **AI 요약 요청**
   - 저장된 활동 경험을 바탕으로 AI 분석을 요청합니다.
   - AI는 경험 요약, 핵심 역량, 주요 성과, 활용 가능한 키워드를 정리합니다.

4. **역량 태그 확인**
   - 사용자는 AI가 정리한 역량 태그, 성과, 키워드를 확인하고 자신의 경험을 다시 이해합니다.
   - 역량 태그는 사용자의 경험을 자기소개서, 포트폴리오, 면접에서 활용하기 쉬운 언어로 변환한 결과입니다.
   - 대표 예시는 다음과 같습니다.
     - 문제 해결력: 문제를 발견하고 원인을 분석해 해결한 경험
     - 협업/커뮤니케이션: 팀원과 역할을 나누고 의견을 조율한 경험
     - 실행력/주도성: 목표를 정하고 실제 결과물까지 만들어낸 경험
     - 분석력/데이터 활용력: 자료, 피드백, 수치, 사용자 반응 등을 바탕으로 판단한 경험

5. **활용 목적 입력**
   - 사용자는 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문 등을 입력합니다.

6. **AI 경험 추천 및 문장 생성**
   - AI는 1차 MVP 기준으로 저장된 경험 중 활용 목적에 가장 적합한 경험 1개만 추천합니다.
   - 추천 결과에는 추천 경험, 추천 이유, 관련 역량 태그, 활용 방향이 포함됩니다.
   - 사용자는 추천 결과를 바탕으로 자기소개서 초안, 포트폴리오 문장, 면접 답변 참고 문장을 확인합니다.

7. **복사 또는 저장**
   - 사용자는 AI가 제안한 참고 문장을 복사해 실제 지원 준비에 활용합니다.
   - 추천 결과 저장은 MVP에 포함하지만, 첫 구현에서는 복사 기능보다 낮은 2순위로 다룹니다.

## Core Flow

```text
활동 기록 입력
→ AI 요약
→ 역량 태그 확인
→ 포트폴리오 / 자소서 / 면접 질문 입력
→ 가장 적합한 경험 1개 추천
→ 참고 문장 생성
→ 복사 / 저장
```

## MVP Excluded Flow

1차 MVP에서는 아래 흐름을 포함하지 않습니다.

- 로그인 / 회원가입
- 학교 인증
- 결제
- 커뮤니티
- GitHub 자동 연동
- 블로그 / Notion 자동 연동
- PDF 포트폴리오 자동 생성
- 고급 추천 알고리즘
- 여러 추천 후보 탐색 (1차 MVP 제외, 2차 MVP 확장 후보)
- 모바일 앱 전용 흐름
- 로그인 기반 사용자별 활동 기록 저장
- 수료증 / 활동사진 등 첨부파일 저장
- Supabase Auth / Postgres / Storage 기반 저장 흐름
- Spring Boot 백엔드 기반 회원 데이터 관리
- MySQL 데이터베이스 저장 흐름

## Future Flow

- 2차 MVP에서는 Supabase Auth / Postgres / Storage를 활용한 로그인 기반 사용자별 저장과 수료증 / 활동사진 첨부 흐름을 검토합니다.
- 2차 MVP에서는 사용자의 활용 목적에 따라 적합한 경험을 1순위, 2순위, 3순위까지 추천하고, 각 경험의 추천 이유를 비교해 보여주는 흐름을 검토합니다.
- 3차 확장에서는 UNIKER 이후 Spring Boot REST API, MySQL 또는 AWS RDS, AWS S3 기반 백엔드 확장 흐름을 검토합니다.
