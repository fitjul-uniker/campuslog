# CampusLog IA

## Purpose

이 문서는 CampusLog 1차 MVP의 메뉴 구조, 페이지 구조, 화면 간 이동 관계를 정리합니다. 1차 MVP에서는 로그인 없이 브라우저 localStorage를 사용해 핵심 사용자 경험을 검증합니다.

## MVP Menu Structure

```text
CampusLog
├── 메인 화면
│   ├── 활동 경험 목록 진입
│   ├── 새 활동 경험 작성 진입
│   └── AI 경험 추천 및 활용 진입
├── 활동 경험 목록 화면
│   └── 활동 경험 상세 진입
├── 활동 경험 작성 / 수정 화면
│   └── 경험 입력 / 수정 후 활동 경험 상세로 이동
├── 활동 경험 상세 화면
│   ├── 원본 내용 확인
│   ├── 수정 진입
│   ├── AI 분석 요청
│   └── 저장된 분석 결과 확인
├── AI 경험 분석 결과 화면
│   └── 특정 활동 경험에 연결된 분석 결과 확인
└── AI 경험 추천 및 활용 화면
    └── 저장된 경험 전체와 분석 결과 기준 가장 적합한 경험 1개 추천
```

## Page Structure

### 1. 메인 화면

- 메인 화면은 1차 MVP의 시작점이자 메인 허브입니다.
- 활동 경험 목록 화면으로 이동할 수 있습니다.
- 새 활동 경험 작성 화면으로 이동할 수 있습니다.
- AI 경험 추천 및 활용 화면으로 진입할 수 있습니다.

### 2. 활동 경험 목록 화면

- 사용자가 localStorage에 저장한 활동 경험 목록을 확인합니다.
- 새 경험 작성 화면으로 이동할 수 있습니다.
- 특정 경험을 선택하면 활동 경험 상세 화면으로 이동합니다.
- 저장된 분석 결과가 있는 경험은 분석 완료 상태를 확인할 수 있습니다.

### 3. 활동 경험 작성 / 수정

- 제목, 기간, 역할, 내용, 성과, 관련 링크를 입력하거나 수정합니다.
- 작성 / 수정 화면은 경험 입력과 수정에 집중하며, AI 분석 요청을 바로 실행하지 않습니다.
- 저장 후 활동 경험 상세 화면으로 이동합니다.
- Primary CTA는 "저장"으로 설정합니다.
- Secondary CTA는 "취소" 또는 "활동 경험 목록으로 돌아가기"로 설정합니다.

### 4. 활동 경험 상세

- 저장된 활동 경험의 원본 내용을 확인합니다.
- 사용자는 활동 경험을 수정하거나, 해당 경험을 기준으로 AI 분석 요청을 실행할 수 있습니다.
- 저장된 분석 결과가 있으면 AI 경험 분석 결과 화면으로 이동할 수 있습니다.
- AI 분석 요청은 활동 경험 상세 화면에서 실행하는 흐름으로 통일합니다.

### 5. AI 경험 분석 결과

- 선택한 특정 활동 경험에 연결된 분석 결과를 확인합니다.
- 분석 결과는 localStorage에 저장되어 사용자가 나중에 다시 확인할 수 있도록 합니다.
- 저장되는 분석 결과에는 경험 요약, 핵심 역량 태그, 주요 성과, 활용 가능한 키워드, 분석 생성일이 포함됩니다.
- 역량 태그는 고정 분류가 아니라, 대표 예시를 참고해 AI가 생성하는 분석 결과입니다.
- 대표 예시는 문제 해결력(문제를 발견하고 원인을 분석해 해결한 경험), 협업/커뮤니케이션(팀원과 역할을 나누고 의견을 조율한 경험), 실행력/주도성(목표를 정하고 실제 결과물까지 만들어낸 경험), 분석력/데이터 활용력(자료, 피드백, 수치, 사용자 반응 등을 바탕으로 판단한 경험)입니다.
- Primary CTA는 "활동 경험 상세로 돌아가기" 또는 "활동 경험 목록으로 돌아가기"로 설정합니다.
- "AI 경험 추천 및 활용"으로 이동하는 버튼은 필요할 경우 Secondary CTA 또는 보조 버튼으로만 둡니다.

### 6. AI 경험 추천 및 활용

- 사용자는 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문 등을 입력합니다.
- 이 화면은 특정 경험 하나의 분석 결과에서 반드시 이어지는 화면이 아닙니다.
- AI는 저장된 경험 전체와 저장된 분석 결과를 참고해 가장 적합한 경험 1개를 추천합니다.
- 1차 MVP에서는 추천 경험을 1개만 제공합니다.
- 추천 결과에는 추천 경험, 추천 이유, 관련 역량 태그, 강조할 성과, 활용 방향, 자기소개서 / 포트폴리오 / 면접 답변 참고 문장이 포함됩니다.
- 추천 결과를 복사할 수 있습니다.
- 추천 결과 저장은 MVP에 포함하지만, 첫 구현에서는 복사 기능보다 낮은 우선순위로 다룹니다.

## Navigation Flow

### 경험 작성 흐름

```text
메인 화면
→ 새 활동 경험 작성
→ 저장
→ 활동 경험 상세
→ AI 분석 요청
→ AI 경험 분석 결과
```

### 경험 기록 / 분석 흐름

```text
메인 화면
→ 활동 경험 목록
→ 활동 경험 상세
→ AI 경험 분석 결과
```

### 활용 추천 흐름

```text
메인 화면
→ AI 경험 추천 및 활용
→ 자기소개서 문항 / 포트폴리오 목적 / 면접 질문 입력
→ 저장된 경험 전체와 분석 결과를 기준으로 가장 적합한 경험 1개 추천
→ 참고 문장 복사 / 저장
```

## Page Relationship

- 메인 화면은 주요 기능으로 진입할 수 있는 허브입니다.
- 활동 경험 목록 화면은 저장된 경험을 확인하고 활동 경험 상세 화면으로 이동하는 화면입니다.
- 활동 경험 작성 / 수정 화면은 경험 원본 데이터를 입력하거나 수정하고, 저장 후 활동 경험 상세 화면으로 이동합니다.
- 활동 경험 상세 화면은 원본 내용을 확인하고 AI 분석 요청을 실행하는 화면입니다.
- AI 경험 분석 결과는 특정 활동 경험에 연결되어 저장되며, 이후 다시 확인할 수 있고 경험 추천의 기준 데이터가 됩니다.
- AI 경험 추천 및 활용 화면은 저장된 경험 전체와 분석 결과를 바탕으로 현재 목적에 가장 적합한 경험 1개를 추천합니다.

## Out of Scope IA

1차 MVP에서는 아래 메뉴와 페이지를 만들지 않습니다.

- 로그인 / 회원가입 페이지
- 마이페이지
- 파일 첨부 관리 페이지
- 관리자 페이지
- 결제 페이지
- 커뮤니티 페이지
- 공개 프로필 페이지
- GitHub / 블로그 / Notion 연동 설정 페이지
- PDF 포트폴리오 생성 페이지
- 여러 추천 후보 탐색 페이지
- Supabase 사용자 / 파일 관리 화면
- Spring Boot 백엔드 관리 화면

## Future Expansion IA

2차 MVP에서는 로그인 / 회원가입, 마이페이지, 사용자별 활동 기록, 수료증 / 활동사진 첨부 관리 화면을 확장 IA로 검토할 수 있습니다.

2차 MVP에서는 사용자의 활용 목적에 따라 적합한 경험을 1순위, 2순위, 3순위까지 추천하고, 각 경험의 추천 이유를 비교해 보여주는 UI를 검토합니다.

3차 확장에서는 UNIKER 이후 Spring Boot REST API, MySQL 또는 AWS RDS, AWS S3를 사용하는 백엔드 포트폴리오 확장용 관리 구조를 별도 검토합니다.
