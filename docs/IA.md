# CampusLog IA

## Purpose

이 문서는 CampusLog 1차 MVP의 메뉴 구조, 페이지 구조, 화면 간 이동 관계를 정리합니다. 1차 MVP에서는 로그인 없이 브라우저 localStorage를 사용해 핵심 사용자 경험을 검증합니다.

## MVP Menu Structure

```text
CampusLog
├── 메인/경험 목록 대시보드
│   ├── 저장된 활동 경험 카드 목록
│   ├── 새 활동 경험 작성 진입
│   ├── 활동 경험 상세 진입
│   └── AI 경험 추천 및 활용 진입
├── 활동 경험 작성 / 수정 화면 (입력 폼 화면)
│   ├── 새 작성: 빈 폼에서 시작
│   └── 수정: 기존 저장 데이터 불러와 편집
├── 활동 경험 상세 화면
│   ├── 원본 내용 확인
│   ├── 수정 진입
│   ├── AI 분석 요청
│   └── 저장된 분석 결과 확인
├── AI 경험 분석 결과 화면
│   └── 특정 활동 경험에 연결된 분석 결과, 핵심 역량 태그, 세부 키워드 확인
├── AI 경험 추천 및 활용 화면
│   └── 저장된 경험 전체와 분석 결과 기준 가장 적합한 경험 1개 추천
└── 추천 기록 화면
    └── 저장된 활용 목적 / 질문 / 추천 결과 목록 확인
```

## Page Structure

### 0. 메인/경험 목록 대시보드

- 메인/경험 목록 대시보드는 1차 MVP의 시작점이자 사용자가 저장한 활동 경험 목록을 바로 보여주는 대시보드입니다.
- 서비스 이름과 짧은 서비스 설명을 상단에 보여줍니다.
- 사용자가 localStorage에 저장한 활동 경험 카드 목록을 확인합니다.
- 사용자는 정렬 옵션과 분석 완료된 경험의 AI 생성 주요 키워드 / 역량 태그 필터를 사용할 수 있습니다.
- 특정 경험 카드를 선택하면 활동 경험 상세 화면으로 이동합니다.
- 새 경험 추가 버튼으로 활동 경험 작성 화면에 진입합니다.
- AI 경험 추천 및 활용 화면으로 진입할 수 있습니다.
- 저장된 경험이 없으면 "첫 경험 기록하기" CTA를 보여주고, 클릭 시 활동 경험 작성 화면으로 이동합니다.
- 1차 MVP의 기본 시작점은 메인/경험 목록 대시보드이며, 로그인 / 회원가입 화면은 만들지 않습니다. 서비스 소개만을 위한 전용 랜딩 라우트는 기본 IA에 포함하지 않지만, premium first viewport, hero-like intro, branded interaction은 대시보드 안에서 또는 명시적인 디자인 작업 범위 안에서 사용할 수 있습니다.

### 1. 활동 경험 작성 / 수정 (입력 폼)

- 이 화면은 활동 경험 상세 화면이 아니라, 새 작성과 수정을 위한 공통 입력 폼 화면입니다.
- 새 작성과 수정은 같은 폼 UI를 공유하지만, 진입 경로와 초기 데이터가 다릅니다.
- 새 작성 모드는 메인/경험 목록 대시보드에서 진입하며, 빈 폼에서 시작합니다.
- 수정 모드는 활동 경험 상세 화면의 수정 버튼에서 진입하며, 기존 저장 데이터를 불러와 편집합니다.
- 제목, 기간, 역할, 내용, 성과, 관련 링크를 입력하거나 수정합니다.
- 작성 / 수정 화면은 경험 입력과 수정에 집중하며, AI 분석 요청을 바로 실행하지 않습니다.
- 저장 후 활동 경험 상세 화면으로 이동합니다.
- Primary CTA는 "저장"으로 설정합니다.
- Secondary CTA는 "취소" 또는 "메인/경험 목록 대시보드로 돌아가기"로 설정합니다.

### 2. 활동 경험 상세

- 저장된 활동 경험의 원본 내용을 확인합니다.
- 사용자는 수정 버튼을 통해 활동 경험 수정 화면으로 이동하거나, 해당 경험을 기준으로 AI 분석 요청을 실행할 수 있습니다.
- 저장된 분석 결과가 있으면 AI 경험 분석 결과 화면으로 이동할 수 있습니다.
- AI 분석 요청은 활동 경험 상세 화면에서 실행하는 흐름으로 통일합니다.
- 메인/경험 목록 대시보드로 돌아갈 수 있습니다.

### 3. AI 경험 분석 결과

- 선택한 특정 활동 경험에 연결된 분석 결과를 확인합니다.
- 분석 결과는 localStorage에 저장되어 사용자가 나중에 다시 확인할 수 있도록 합니다.
- 저장되는 분석 결과에는 경험 요약, 핵심 역량 태그, 주요 성과, 세부 키워드 또는 활용 가능한 키워드, 분석 생성일이 포함됩니다.
- 핵심 역량 태그는 사용자가 직접 고정 목록에서 선택하는 값이 아니라, 대표 예시를 참고해 AI가 활동 내용을 분석해 생성하는 결과입니다.
- 핵심 역량 태그는 자기소개서, 면접, 포트폴리오 문장에 활용하기 좋은 넓은 역량 범주로 보여줍니다.
- 대표 예시는 문제 해결력(문제를 발견하고 원인을 분석해 해결한 경험), 협업/커뮤니케이션(팀원과 역할을 나누고 의견을 조율한 경험), 실행력/주도성(목표를 정하고 실제 결과물까지 만들어낸 경험), 분석력/데이터 활용력(자료, 피드백, 수치, 사용자 반응 등을 바탕으로 판단한 경험)입니다.
- 세부 키워드는 AI 개발, Python, Git 협업, 데이터 구조화, MVP 개발처럼 포트폴리오 구성과 추천 정확도에 활용하기 좋은 구체 기술·활동 키워드입니다.
- Primary CTA는 "활동 경험 상세로 돌아가기" 또는 "메인/경험 목록 대시보드로 돌아가기"로 설정합니다.
- "AI 경험 추천 및 활용"으로 이동하는 버튼은 필요할 경우 Secondary CTA 또는 보조 버튼으로만 둡니다.

### 4. AI 경험 추천 및 활용

- 사용자는 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문 등을 입력합니다.
- 이 화면은 특정 경험 하나의 분석 결과에서 반드시 이어지는 화면이 아닙니다.
- AI는 저장된 경험 전체와 저장된 분석 결과를 참고해 가장 적합한 경험 1개를 추천합니다.
- 1차 MVP에서는 추천 경험을 1개만 제공합니다.
- 추천 결과에는 추천 경험, 추천 이유, 관련 핵심 역량 태그, 강조할 성과, 활용 방향, 자기소개서 / 포트폴리오 / 면접 답변 참고 문장이 포함됩니다.
- 추천 결과를 복사할 수 있습니다.
- 추천 결과 저장은 MVP에 포함하지만, 첫 구현에서는 복사 기능보다 낮은 우선순위로 다룹니다.

### 5. 추천 기록

- 사용자는 왼쪽 내비게이션의 추천 기록 메뉴에서 저장된 추천 결과 목록을 확인합니다.
- 추천 기록 목록에는 활용 목적, 질문 / 문항, 추천 경험, 추천 생성일을 보여줍니다.
- 사용자가 특정 추천 기록을 선택하면 해당 활용 목적, 질문, 추천 이유, 관련 태그, 강조할 성과, 활용 방향, 참고 문장을 다시 확인할 수 있습니다.
- 추천 기록은 `campuslog:v1:recommendations`에 저장된 localStorage 데이터를 기준으로 보여줍니다.

## Navigation Flow

### 경험 새 작성 흐름

```text
메인/경험 목록 대시보드
→ 새 경험 추가 또는 첫 경험 기록하기
→ 활동 경험 작성 화면 (빈 폼)
→ 저장
→ 활동 경험 상세
→ AI 분석 요청
→ AI 경험 분석 결과
```

### 경험 수정 흐름

```text
활동 경험 상세
→ 수정 버튼
→ 활동 경험 수정 화면 (기존 저장 데이터 불러오기)
→ 저장
→ 활동 경험 상세
→ AI 분석 요청
→ AI 경험 분석 결과
```

### 경험 기록 / 분석 흐름

```text
메인/경험 목록 대시보드
→ 기존 경험 카드 선택
→ 활동 경험 상세
→ AI 분석 요청
→ AI 경험 분석 결과
```

### 빈 상태 흐름

```text
메인/경험 목록 대시보드
→ 저장된 경험 없음
→ 첫 경험 기록하기
→ 활동 경험 작성 화면 (빈 폼)
```

### 활용 추천 흐름

```text
메인/경험 목록 대시보드
→ AI 경험 추천 및 활용
→ 자기소개서 문항 / 포트폴리오 목적 / 면접 질문 입력
→ 저장된 경험 전체와 분석 결과를 기준으로 가장 적합한 경험 1개 추천
→ 참고 문장 복사 / 저장
→ 추천 기록에서 저장된 질문과 추천 결과 다시 확인
```

## Page Relationship

- 메인/경험 목록 대시보드는 주요 기능 진입점이자 저장된 경험 목록을 바로 확인하는 첫 화면입니다.
- 1차 MVP에서는 별도 허브 화면이나 목록 전용 화면을 두지 않습니다.
- 메인/경험 목록 대시보드에서 경험 카드를 선택하면 활동 경험 상세 화면으로 이동하고, 새 경험 추가 또는 첫 경험 기록하기 CTA를 선택하면 활동 경험 작성 화면으로 이동합니다.
- 활동 경험 작성 / 수정 화면은 활동 경험 상세 화면이 아니라 경험 원본 데이터를 입력하거나 수정하는 공통 입력 폼 화면입니다.
- 새 작성 모드는 빈 폼에서 시작하고, 수정 모드는 기존 저장 데이터를 불러온 상태에서 시작합니다.
- 활동 경험 상세 화면은 저장된 원본 내용을 확인하고 수정 진입 또는 AI 분석 요청을 실행하는 화면입니다.
- AI 경험 분석 결과는 특정 활동 경험에 연결되어 저장되며, 이후 다시 확인할 수 있고 경험 추천의 기준 데이터가 됩니다.
- AI 경험 분석 결과의 핵심 역량 태그는 넓은 역량 범주이고, 세부 키워드는 구체 기술·활동 키워드입니다.
- AI 경험 추천 및 활용 화면은 저장된 경험 전체와 분석 결과를 바탕으로 현재 목적에 가장 적합한 경험 1개를 추천합니다.
- 추천 기록 화면은 AI 경험 추천 및 활용 화면에서 저장한 활용 목적 / 질문 / 추천 결과를 목록으로 다시 확인하는 별도 화면입니다.

## Out of Scope IA

1차 MVP에서는 아래 메뉴와 페이지를 만들지 않습니다.

- 로그인 / 회원가입 페이지
- 로그인 / 회원 전환 목적의 별도 서비스 소개 랜딩 페이지
- 마이페이지
- 파일 첨부 관리 페이지
- 관리자 페이지
- 결제 페이지
- 커뮤니티 페이지
- 공개 프로필 페이지
- GitHub / 블로그 / Notion 연동 설정 페이지
- PDF 포트폴리오 생성 페이지
- 여러 추천 후보 탐색 페이지
- 역량 태그 모아보기 페이지
- Supabase 사용자 / 파일 관리 화면
- UNIKER 이후 개인 Backend Portfolio Edition 관리 화면

## Future Expansion IA

2차 MVP에서는 Next.js Full Stack + Supabase 기반으로 외부 사용자 전환용 서비스 소개 랜딩, 로그인 / 회원가입, 마이페이지, 사용자별 활동 기록, 수료증 / 활동사진 첨부 관리 화면을 확장 IA로 검토할 수 있습니다.

2차 MVP에서는 사용자의 활용 목적에 따라 적합한 경험을 1순위, 2순위, 3순위까지 추천하고, 각 경험의 추천 이유를 비교해 보여주는 UI를 검토합니다.

2차 이후 확장에서는 사용자가 지금까지 받은 핵심 역량 태그를 한 번에 확인하는 역량 태그 모아보기 화면을 검토할 수 있습니다.
이 화면에서는 특정 역량 태그를 클릭해 연결된 활동 경험 목록을 확인하고, 해당 역량이 자기소개서, 포트폴리오, 면접에서 어떻게 활용될 수 있는지 추천할 수 있습니다.
장기적으로는 사용자가 자주 쌓은 역량과 부족한 역량을 보고 앞으로 어떤 활동을 하면 좋을지 방향성을 얻는 화면으로 확장할 수 있습니다.

UNIKER 이후 개인 Backend Portfolio Edition에서는 Spring Boot REST API, MySQL 또는 AWS RDS, AWS S3를 사용하는 백엔드 포트폴리오 확장용 관리 구조를 별도 검토합니다.
