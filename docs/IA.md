# CampusLog IA

## Purpose

이 문서는 CampusLog 1차 MVP의 메뉴 구조, 페이지 구조, 화면 간 이동 관계를 정리합니다. 1차 MVP에서는 로그인 없이 브라우저 localStorage를 사용해 핵심 사용자 경험을 검증합니다.

## MVP Menu Structure

```text
CampusLog
├── 메인 화면 / 활동 경험 목록
│   ├── 활동 경험 목록 확인
│   ├── 새 활동 경험 작성
│   ├── 활동 경험 상세 / 수정
│   ├── 저장된 AI 분석 결과 확인
│   └── AI 경험 추천 및 활용 진입
├── AI 분석
│   └── 경험 요약 / 역량 / 성과 / 키워드 확인
└── 활용 지원
    └── 자기소개서 / 포트폴리오 / 면접 질문 기반 경험 1개 추천
```

## Page Structure

### 1. 메인 화면 / 활동 경험 목록

- 활동 경험 목록 화면은 1차 MVP의 시작점이자 메인 허브입니다.
- 사용자가 localStorage에 저장한 활동 경험 목록을 확인합니다.
- 새 경험 작성 화면으로 이동할 수 있습니다.
- 특정 경험을 선택하면 상세 / 수정 화면으로 이동합니다.
- 저장된 AI 분석 결과를 다시 확인할 수 있습니다.
- AI 경험 추천 및 활용 화면으로 진입할 수 있습니다.

### 2. 활동 경험 작성 / 수정

- 제목, 기간, 역할, 내용, 성과, 관련 링크를 입력하거나 수정합니다.
- 저장 후 Primary CTA는 "AI 분석하기"로 설정합니다.
- Secondary CTA는 "메인 화면 / 활동 경험 목록으로 돌아가기"로 설정합니다.
- 사용자는 저장 후 바로 AI 분석으로 이동하거나 메인 화면으로 돌아갈 수 있습니다.

### 3. AI 경험 분석 결과

- 선택한 경험에 대한 AI 요약, 핵심 역량, 주요 성과, 키워드를 확인합니다.
- AI 분석 결과는 localStorage에 저장되어 사용자가 나중에 다시 확인할 수 있도록 합니다.
- 저장되는 분석 결과에는 경험 요약, 역량 태그, 주요 성과, 활용 가능한 키워드, 분석 생성일이 포함됩니다.
- 분석 결과 확인 후 Primary CTA는 "활용 지원으로 이동"으로 설정합니다.
- Secondary CTA는 "메인 화면 / 활동 경험 목록으로 돌아가기"로 설정합니다.

### 4. AI 경험 추천 및 활용

- 사용자는 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문 등을 입력합니다.
- AI는 저장된 경험 전체와 저장된 분석 결과를 참고해 가장 적합한 경험 1개를 추천합니다.
- 1차 MVP에서는 추천 경험을 1개만 제공합니다.
- 추천 결과에는 추천 경험, 추천 이유, 관련 역량 태그, 활용 방향, 자기소개서 / 포트폴리오 / 면접 답변 참고 문장이 포함됩니다.
- 추천 결과를 복사할 수 있습니다.
- 추천 결과 저장은 MVP에 포함하지만, 첫 구현에서는 복사 기능보다 낮은 2순위로 다룹니다.

## Navigation Flow

### Primary Flow

```text
메인 화면 / 활동 경험 목록
→ 활동 경험 작성 / 수정
→ AI 경험 분석 결과 저장
→ AI 경험 추천 및 활용
→ 추천 결과 복사 / 저장
```

### Hub Access Flow

```text
메인 화면
→ 활동 경험 목록 확인
→ 새 활동 경험 작성
→ 저장된 AI 분석 결과 확인
→ AI 경험 추천 및 활용 진입
```

## Page Relationship

- 메인 화면 / 활동 경험 목록은 모든 기능으로 진입할 수 있는 허브입니다.
- 활동 경험 작성 / 수정 화면은 AI 분석의 입력 데이터를 만듭니다.
- AI 경험 분석 결과는 저장되어 이후 다시 확인할 수 있으며, 경험 추천의 기준 데이터가 됩니다.
- AI 경험 추천 및 활용 화면은 저장된 경험 전체와 분석 결과를 바탕으로 현재 목적에 가장 적합한 경험 1개를 추천합니다.

## Out of Scope IA

1차 MVP에서는 아래 메뉴와 페이지를 만들지 않습니다.

- 로그인 / 회원가입 페이지
- 마이페이지
- 파일 첨부 관리 페이지
- 관리자 페이지
- 결제 페이지
- 커뮤니티 페이지
- 공개 프로필 페이지
- GitHub / 블로그 / Notion 연동 설정 페이지
- PDF 포트폴리오 생성 페이지
- 여러 추천 후보 탐색 페이지
- Supabase 사용자 / 파일 관리 화면
- Spring Boot 백엔드 관리 화면

## Future Expansion IA

2차 MVP에서는 로그인 / 회원가입, 마이페이지, 사용자별 활동 기록, 수료증 / 활동사진 첨부 관리 화면을 확장 IA로 검토할 수 있습니다.

2차 MVP에서는 사용자의 활용 목적에 따라 적합한 경험을 1순위, 2순위, 3순위까지 추천하고, 각 경험의 추천 이유를 비교해 보여주는 흐름을 검토합니다.

3차 확장에서는 UNIKER 이후 Spring Boot REST API, MySQL 또는 AWS RDS, AWS S3를 사용하는 백엔드 포트폴리오 확장용 관리 구조를 별도 검토합니다.
