# CampusLog IA

## Purpose

이 문서는 CampusLog MVP의 메뉴 구조, 페이지 구조, 화면 간 이동 관계를 정리합니다. MVP에서는 로그인 없이 브라우저 LocalStorage를 사용해 핵심 사용자 경험을 검증합니다.

## MVP Menu Structure

```text
CampusLog
├── 활동 경험
│   ├── 활동 경험 목록
│   ├── 활동 경험 작성
│   └── 활동 경험 상세 / 수정
├── AI 분석
│   └── 경험 요약 / 역량 / 성과 / 키워드 확인
└── 활용 지원
    └── 자기소개서 / 포트폴리오 / 면접 질문 기반 경험 1개 추천
```

## Page Structure

### 1. 활동 경험 목록

- 사용자가 저장한 활동 경험을 확인하는 시작 화면입니다.
- 새 경험 작성 화면으로 이동할 수 있습니다.
- 특정 경험을 선택하면 상세 / 수정 화면으로 이동합니다.

### 2. 활동 경험 작성 / 수정

- 제목, 기간, 역할, 내용, 성과, 관련 링크를 입력하거나 수정합니다.
- 저장 후 활동 경험 목록 또는 AI 분석 화면으로 이동할 수 있습니다.

### 3. AI 경험 분석 결과

- 선택한 경험에 대한 AI 요약, 핵심 역량, 주요 성과, 키워드를 확인합니다.
- 분석 결과를 확인한 뒤 활용 지원 화면으로 이동할 수 있습니다.

### 4. AI 경험 추천 및 활용

- 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문 등을 입력합니다.
- AI가 가장 적합한 경험 1개, 추천 이유, 활용 방향, 참고 문장을 보여줍니다.
- 추천 결과를 복사할 수 있습니다.
- 추천 결과 저장은 MVP에 포함하지만, 첫 구현에서는 복사 기능보다 낮은 2순위로 다룹니다.

## Navigation Flow

```text
활동 경험 목록
→ 활동 경험 작성 / 수정
→ AI 경험 분석 결과
→ AI 경험 추천 및 활용
→ 추천 결과 복사 / 저장
```

## Page Relationship

- 활동 경험 목록은 MVP의 시작점입니다.
- 활동 경험 작성 / 수정 화면은 AI 분석의 입력 데이터를 만듭니다.
- AI 경험 분석 결과는 경험 추천의 기준 데이터가 됩니다.
- AI 경험 추천 및 활용 화면은 사용자의 현재 목적에 맞는 경험과 참고 문장을 제공합니다.

## Out of Scope IA

이번 MVP에서는 아래 메뉴와 페이지를 만들지 않습니다.

- 로그인 / 회원가입 페이지
- 마이페이지
- 관리자 페이지
- 결제 페이지
- 커뮤니티 페이지
- 공개 프로필 페이지
- GitHub / 블로그 / Notion 연동 설정 페이지
- PDF 포트폴리오 생성 페이지
- 여러 추천 후보 탐색 페이지
- Spring Boot 백엔드 관리 화면
