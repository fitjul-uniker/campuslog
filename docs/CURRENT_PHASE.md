## v1.1 완료 기준선

v1.1에서 아래 흐름을 완성했습니다.

```text
활동 추가
→ 날짜별로 실제 한 일 기록
→ 월간 캘린더에서 기록 확인
→ 활동 종료
```

## 2차 MVP 목표

2차 MVP는 v1.1의 핵심 흐름을 외부 사용자가 계정 기반으로 안전하게 사용할 수 있도록 확장하고, AI 결과의 품질과 전체 사용자 경험을 고도화하는 단계입니다.

### Track A. 인증·데이터·AI 고도화

담당: 다른 팀원

- 이메일 또는 이에 준하는 아이디 + 비밀번호 로그인, 회원가입, 로그아웃, 세션 유지
- Google OAuth 로그인
- 이메일·Google 회원가입의 이름·닉네임 입력과 OAuth 후 온보딩 복귀
- 사용자별 데이터 저장과 접근 제어
- Supabase Auth / Postgres 도입
- 파일 저장이 실제 기능에 포함될 때만 Supabase Storage 도입
- 기존 localStorage 데이터의 사용자 계정 마이그레이션 정책
- AI 분석 품질, 결과 구조, 재분석 흐름 고도화
- AI 추천 정확도, 근거 설명, 활용 결과 고도화
- JD / 직무 요구사항 / 우대사항 원문 기반 경험 추천과 부족 경험 비교
- 질문 이미지 OCR / vision 입력과 답변 초안 생성
- API 입력 검증, 인증, rate limit, 비용 한도와 오류 관측성 강화

다음 AI 개발 순서:

1. AI 경험 분석 v2: STAR, 원본 근거, 부족한 정보, 자소서 소재 각도를 추가합니다. 구현 완료.
2. 추천 v2: 자기소개서 문항 / JD 요구사항을 추출하고 경험 Top 3, 부족 근거, 선택 이유를 비교합니다. 구현 완료.
3. 답변 초안 생성: 사용자가 선택한 500자 / 800자 / 1000자 자기소개서, 면접 답변, 포트폴리오 설명 초안을 근거 기반으로 제공합니다. 구현 완료.
4. 기록 보완 루프: AI가 부족한 정보를 질문하고 사용자가 답하면 분석과 추천을 재생성합니다. 구현 완료.
5. OCR / JD 이미지 입력: 텍스트 붙여넣기 흐름 안정화 후 원본 저장 없는 일회성 입력으로 붙입니다.

- 로딩, 빈 상태, 오류, 저장 성공, 네트워크 복구 상태 개선
- 과한 색상, 그라디언트, 반복 애니메이션 없이 미니멀한 SaaS 완성도 유지

## 공동 계약

두 Track은 아래 계약을 함께 지킵니다.

- 기존 v1.1 localStorage 데이터는 자동 이전하거나 자동 삭제하지 않습니다.
- 활동 종료 AI 합성 초안과 상태도 사용자 소유 데이터로 취급하며, 인증·RLS·호출 제한·완료 경험 생성 멱등성 계약을 적용합니다.
- 인증·DB schema와 화면 상태는 같은 PR 또는 연결된 PR에서 문서화합니다.
- 디자인 Track은 API 계약을 임의로 변경하지 않고, 데이터·AI Track은 승인된 사용자 흐름을 임의로 바꾸지 않습니다.
- 충돌이 있으면 `docs/CURRENT_PHASE.md` → `PRD.md` → 사용자 흐름·IA·화면 명세 순으로 판단합니다.
- AI가 만든 결과는 사용자 원본 기록과 구분하고, 근거·실패·재시도 상태를 명확히 표시합니다.
- API Key와 관리자 권한은 서버 환경에서만 사용하며 브라우저에 노출하지 않습니다.

## 2차 MVP 포함 범위

- 인증과 사용자별 데이터
- v1.1 localStorage 데이터의 명확한 유지 정책. 이전 기능은 실제 기존 사용자 데이터 보존 요구가 생길 때 선택적으로 진행
- 현재 활동·경험·분석·추천 기능의 DB 연결
- AI 분석 및 추천 품질 고도화
- 디자인 시스템과 주요 사용자 흐름 고도화
- 공개 배포를 위한 보안, 비용, 오류 처리, 접근성 검증

## 현재 제외 범위

아래 항목은 별도 승인이나 새로운 단계 선언 없이 2차 MVP에 추가하지 않습니다.

- 결제와 유료 플랜
- 커뮤니티와 팀 협업
- 관리자 운영 도구 전체
- 모바일 네이티브 앱
- 공개 프로필과 소셜 기능
- GitHub, 블로그, Notion 자동 동기화
- PDF 포트폴리오 자동 생성
- Spring Boot / MySQL / AWS 기반 개인 Backend Portfolio Edition

## 완료 조건

- 이메일 또는 이에 준하는 아이디 + 비밀번호 인증, Google OAuth, 로그아웃과 세션 복구가 동작합니다.
- 이메일 가입은 자격 증명 뒤 이름·닉네임을 저장하고, Google 신규·미완료 계정은 OAuth 시작 위치와 무관하게 같은 프로필 단계를 완료한 후 안전한 목적지로 복귀합니다.
- 사용자 A와 사용자 B의 데이터가 분리됩니다.
- v1.1의 활동·일일 기록·완료 경험·분석·추천 흐름이 DB 연결 후에도 유지됩니다.
- 기존 localStorage 데이터를 로그인 계정의 기본 데이터로 자동 표시하지 않고, 원본을 자동 삭제하지 않는 정책이 구현과 화면에서 일치합니다.
- AI API가 로그인 사용자에게만 열리고, 호출 제한과 비용 방어 정책이 적용됩니다.
- AI 분석과 추천 고도화 결과가 근거, 오류, 재시도 상태를 포함합니다.
- JD 원문·질문 이미지 입력에서도 요구사항 추출, 경험 추천, 부족 경험 비교, 답변 초안이 근거 기반으로 동작합니다.
- 핵심 화면이 데스크톱과 모바일에서 가로 잘림 없이 동작합니다.
- 키보드 접근성, 텍스트 대비, reduced motion, loading / empty / error 상태를 확인합니다.
- 인증 우회, 다른 사용자 데이터 접근, API Key 노출, 무제한 AI 호출 위험을 점검합니다.

## 범위 변경 규칙

- 사용자가 현재 작업에서 명시적으로 승인한 범위 변경은 1차 MVP 제한보다 우선합니다.
- 승인된 변경은 같은 작업에서 `CURRENT_PHASE.md`, `PRD.md`, 관련 화면 문서 또는 `ISSUE_LOG.md`에 기록합니다.
- 보안, 개인정보, 비용, 외부 배포, 데이터 삭제처럼 되돌리기 어려운 결정은 구현 전에 팀 확인을 받습니다.
