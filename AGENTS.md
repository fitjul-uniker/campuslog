# AGENTS.md

## Product

CampusLog는 대학생이 프로젝트, 공모전, 인턴, 대외활동 등 다양한 활동 경험을 기록하는 서비스입니다. AI는 기록된 경험을 분석하고, 자기소개서·포트폴리오·면접 준비 상황에 맞는 경험을 추천합니다.

이 프로젝트는 커널아카데미 대학생 AI 크루 UNIKER 1기 팀 핏줄의 10주 MVP 프로젝트입니다. 많은 기능보다, 경험 기록부터 AI 분석과 추천까지 이어지는 핵심 흐름을 우선합니다.

## Source of truth

Codex는 작업 전에 아래 순서로 프로젝트 기준을 확인합니다.

1. `PRD.md`: MVP 범위, 제외 기능, 성공 기준
2. `docs/USER_FLOW.md`: 사용자의 핵심 흐름
3. `docs/IA.md`: 메뉴 구조와 페이지 이동 관계
4. `docs/SCREEN_SPEC.md`: 화면별 UI 요소, 입력값, 출력값, 버튼 동작
5. `README.md`: 프로젝트 소개, 기술 방향, 실행 방법
6. 현재 코드와 데이터: 이미 구현된 흐름을 깨지 않기 위해 확인

문서와 코드가 다를 경우 아래 우선순위로 판단합니다.

`PRD.md` > `docs/USER_FLOW.md` / `docs/IA.md` / `docs/SCREEN_SPEC.md` > `README.md` > 현재 코드와 데이터

기능 범위 판단은 항상 `PRD.md`를 우선합니다. 새 기능이나 화면을 제안하기 전에도 먼저 `PRD.md`의 MVP 범위와 제외 기능을 확인합니다.

## Working Rules

Codex는 모든 작업을 아래 순서와 원칙에 따라 진행합니다.

- 작업 전 관련 기준 문서를 먼저 읽습니다.
- 바로 구현하지 않고 변경 계획을 먼저 세웁니다.
- 사용자가 요청한 범위만 수정합니다.
- 한 번에 큰 리팩토링을 하지 않습니다.
- 기능 하나 또는 문서 하나 단위로 작게 작업합니다.
- 기존 사용자 흐름을 깨지 않습니다.
- 작업 후 수정한 파일, 변경 내용, 확인한 내용을 요약합니다.
- 작업이 끝나면 다음 작업자가 이어서 볼 수 있게 남은 작업과 주의사항을 정리합니다.

## Build and verify

작업 후에는 현재 MVP 기준으로 아래 흐름이 유지되는지 확인합니다.

- 사용자가 활동 경험을 입력할 수 있는가
- 입력한 경험을 다시 확인할 수 있는가
- AI가 경험을 요약하고 핵심 역량, 주요 성과, 키워드를 정리할 수 있는가
- 사용자가 자기소개서 문항, 포트폴리오 작성 목적, 면접 질문 등을 입력할 수 있는가
- AI가 저장된 경험 중 적절한 경험을 추천할 수 있는가
- AI가 추천 이유와 활용 방향을 함께 제공하는가
- 로그인, 회원가입, Spring Boot 백엔드 없이도 MVP 핵심 흐름이 동작하는가
- 변경 후 수정한 파일과 확인한 내용을 간단히 요약하는가

현재는 문서와 초기 구조를 준비하는 단계입니다. 구현 작업 전에는 `PRD.md`, `docs/USER_FLOW.md`, `docs/IA.md`, `docs/SCREEN_SPEC.md`, `README.md`의 MVP 범위, 핵심 기능, 제외 기능이 서로 일치하는지도 확인합니다.

## Guardrails

보안과 개인정보 보호를 위해 아래 규칙을 지킵니다.

- API Key, 토큰, 비밀번호, 인증 정보는 문서나 코드에 직접 넣지 않습니다.
- 외부 API를 사용할 경우 API Key는 반드시 환경 변수로 관리하고, 예시 파일에는 실제 값 대신 placeholder만 사용합니다.
- `.env` 파일은 커밋하지 않습니다.
- 실제 사용자 데이터나 민감한 개인정보를 GitHub에 커밋하지 않습니다.
- 샘플 데이터는 공개 가능한 활동 이력만 사용합니다.
- 실제 인물 이름을 사용할 경우에도 전화번호, 이메일, 학번, 주소, 실제 지원서 원문, 민감한 개인정보는 포함하지 않습니다.
- 실제 학교 인증 정보, 시간표, 캘린더, 위치 기록은 샘플 데이터로 사용하지 않습니다.
- 샘플 데이터는 서비스 흐름을 검증하기 위한 가짜 예시 또는 공개 가능한 수준의 활동 기록만 사용합니다.

## Scope control

이번 MVP에서 집중할 기능은 아래 3개입니다.

- 활동 경험 기록
- AI 경험 분석
- AI 경험 추천 및 활용 지원

이번 MVP에서는 아래 기능을 구현하지 않습니다.

- 로그인 / 회원가입
- 학교 인증
- GitHub 자동 연동
- 블로그 / Notion 연동
- PDF 포트폴리오 자동 생성
- 관리자 페이지
- 결제
- 커뮤니티
- 모바일 앱
- 팀 협업 기능
- 공개 프로필
- 복잡한 추천 알고리즘
- Spring Boot 백엔드 구현
- MySQL 데이터베이스 구축

Codex는 사용자가 명시적으로 요청하지 않는 이상 위 제외 기능을 제안하거나 구현하지 않습니다. 새 기능 아이디어가 나오면 바로 구현하지 않고, 먼저 `PRD.md`의 MVP 범위와 제외 기능에 맞는지 확인합니다. MVP에서는 복잡한 추천 알고리즘보다 사용자가 이해할 수 있는 단순한 AI 분석과 추천 흐름을 우선합니다.

## Change Safety

AI가 코드나 문서를 과하게 바꾸지 않도록 아래 규칙을 지킵니다.

- 요청하지 않은 파일은 수정하지 않습니다.
- 기존 내용을 삭제하거나 대체하기 전에는 이유를 설명합니다.
- 전체 코드나 문서를 한 번에 갈아엎지 않습니다.
- 대규모 구조 변경이 필요해 보이면 먼저 제안만 하고, 승인 전에는 실행하지 않습니다.
- 불필요한 dependency 추가를 피합니다.
- 임시 코드, 테스트 코드, 샘플 데이터는 공개 가능한 수준으로만 작성합니다.

## Git and Commit Rules

GitHub 협업을 위해 아래 규칙을 지킵니다.

- 작은 작업 단위로 커밋할 수 있게 변경 범위를 작게 유지합니다.
- 작업 후 커밋 메시지 후보를 제안합니다.
- 커밋 전에는 변경 파일 목록과 변경 이유를 요약합니다.
- `main` 브랜치에 바로 큰 변경을 하지 않습니다.
- AI가 만든 코드도 사람이 검토할 수 있도록 변경 내용을 명확히 설명합니다.

## Output Format

```text
작업 완료 요약:
- 수정한 파일:
- 변경 내용:
- 확인한 기준 문서:
- 검증한 내용:
- 남은 작업:
- 추천 커밋 메시지:
```
